  <name>Task 1: Implement TypeScript emitter, extraction log, and CLI orchestrator</name>
  <files>packages/data-extractor/src/emitters/ts-emitter.ts, packages/data-extractor/src/logging/extraction-log.ts, packages/data-extractor/src/cli.ts</files>
  <action>
1. **TypeScript emitter** (`ts-emitter.ts`): Per D-10 and D-11.
   - `emitTypescriptCatalog(options: EmitOptions): string` where EmitOptions = `{ catalogName: string; schemaImport: string; typeName: string; data: unknown; datasetId: string; }`.
   - Generates a TypeScript source string with:
     ```typescript
     import { ${schemaName} } from '@data-extractor/contracts/${contractFile}';
     import type { ${typeName} } from '@data-extractor/contracts/${contractFile}';

     export const ${catalogName}: ${typeName} = ${schemaName}.parse(${JSON.stringify(data, null, 2)});
     ```
   - Use `JSON.stringify(data, null, 2)` for the data payload. This preserves Spanish characters correctly because JSON.stringify produces valid Unicode.
   - Add a file header comment: `// Generated by data-extractor on ${timestamp}. Do not edit manually.`
   - Per D-12: The datasetId is embedded in the data payload itself, not as a separate field.
   - `writeTypescriptCatalog(outputPath: string, content: string): void` -- writes the file to disk using `fs.writeFileSync`. Creates the output directory if it doesn't exist.

2. **Extraction log** (`extraction-log.ts`): Per D-14.
   - `ExtractionLog` class that accumulates extraction events:
     - `addCatalog(name: string, itemCount: number)`: record a successfully assembled catalog.
     - `addWarning(catalog: string, message: string)`: record a warning.
     - `addSkipped(catalog: string, item: string, reason: string)`: record a skipped item.
     - `setProvenance(datasetId: string, manifestSha1: string, timestamp: string)`: record dataset provenance.
     - `toReport(): string`: produce a human-readable report string with:
       - Header: "Puerta de Baldur Data Extraction Report"
       - Provenance: datasetId, manifestSha1, extraction timestamp
       - Per-catalog summary: name, item count
       - Warnings section (if any)
       - Skipped items section (if any)
       - Footer: total items, total warnings, total skipped

3. **CLI orchestrator** (`cli.ts`): Per D-07, single-pass extraction of ALL catalogs.
   - Import config constants from `./config.ts`.
   - Orchestration sequence:
     a. Open NwsyncReader (metaDb, dataDb, PUERTA_MANIFEST_SHA1).
     b. Open BaseGameReader (BASE_GAME_KEY, BASE_GAME_DIR).
     c. Create TlkResolver from base TLK + custom TLK via nwsync.
     d. Create ExtractionLog.
     e. Generate datasetId: `puerta-ee-${YYYY-MM-DD}+${first8CharsOfManifestSha1}` (per datasetIdSchema pattern).
     f. Run assemblers in order (dependencies flow forward):
        1. `assembleClassCatalog` -> produces classRows needed by skill/feat/spell assemblers
        2. `assembleRaceCatalog`
        3. `assembleSkillCatalog(... classRows)` -> uses classRows for cls_skill mapping
        4. `assembleFeatCatalog(... classRows)` -> uses classRows for cls_feat mapping. Produces featIdsByRow.
        5. `assembleSpellCatalog(... classRows)` -> uses classRows for cls_spgn/spkn mapping. Produces spellIdsByRow.
        6. `assembleDomainCatalog(... featIdsByRow, spellIdsByRow)` -> cross-references
        7. `assembleDeityData`
     g. For each non-null catalog, emit TypeScript file to OUTPUT_DIR using the emitter:
        - `compiled-classes.ts` from ClassCatalog
        - `compiled-races.ts` from RaceCatalog
        - `compiled-skills.ts` from SkillCatalog
        - `compiled-feats.ts` from FeatCatalog
        - `compiled-spells.ts` from SpellCatalog
        - `compiled-domains.ts` from DomainCatalog
        - `compiled-deities.ts` -- if deity catalog is null, emit a placeholder export: `export const compiledDeityCatalog = null; // No deity 2DA found. See extraction log.`
     h. Accumulate warnings from all assemblers into the ExtractionLog.
     i. Write extraction log to `packages/data-extractor/extraction-report.txt`.
     j. Close NwsyncReader.
     k. Print summary to console: catalogs generated, total items, total warnings.
   - Handle errors: catch and report. If a single assembler fails, log the error and continue with other catalogs (best-effort per D-13).
   - The CLI is run as: `cd packages/data-extractor && npx tsx src/cli.ts` or `pnpm extract` from the data-extractor package.
  </action>
