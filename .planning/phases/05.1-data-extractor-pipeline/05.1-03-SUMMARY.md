## Decisions Made
- **Prepared statements for nwsync:** All SQL queries use prepared statements scoped to PUERTA_MANIFEST_SHA1 to avoid mixing Arelith data (T-05.1-09) and ensure clean connection lifecycle (T-05.1-10).
- **Save progressions from table names:** SavingThrowTable references (e.g., CLS_SAVTHR_BARB) are not present in nwsync. Saving throw progressions are derived from a known-class heuristic based on the table naming convention.
- **Race size from appearance.2da:** The racialtypes.2da does not have a SizeCategory column. Size is resolved by looking up the Appearance column index in appearance.2da's SIZECATEGORY column.
- **Spanish slug transliteration:** 2DA Labels like "Engañar" contain accented characters that don't match the canonical ID regex. A shared slug-utils module transliterates Latin-1 accented characters to ASCII before slug generation.
- **Deity gap documentation:** No deity 2DA was found in nwsync or base game (confirmed by listing all resrefs). The assembler returns null with a warning documenting that deity data requires manual overrides.
- **Actual counts differ from plan estimates:** Classes: 41 (not 47); Races: 46 (not ~19); Skills: 39 (matches). The plan's estimates were approximations from the research phase.
