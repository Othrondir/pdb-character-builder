<domain>
## Phase Boundary

Build the nwsync-to-compiled-JSON extraction pipeline that reads Puerta de Baldur server data directly from the local nwsync SQLite databases and produces typed, static TypeScript catalogs for the planner. Covers all character-building data: classes, races, subraces, skills, feats, spells, domains, and deities. This phase produces the data layer — it does not modify planner UI or routing.

<decisions>
## Implementation Decisions

### Data source
- **D-01:** The extractor reads directly from the nwsync SQLite databases at `C:\Users\pzhly\Documents\Neverwinter Nights\nwsync\` (nwsyncmeta.sqlite3 + nwsyncdata_0.sqlite3). No intermediate file step.
- **D-02:** NSYC v3 decompression is integrated into the extractor. Format: 4-byte "NSYC" header + 4-byte version + 16-byte sub-header + zstd payload at offset 24. Use Python `zstandard` or Node.js equivalent.
- **D-03:** The extractor also reads the base game BIF/KEY files at `C:\Program Files (x86)\Steam\steamapps\common\Neverwinter Nights\data\` for base-game data that the server doesn't override.

### TLK resolution
- **D-04:** Extract the Puerta custom TLK (`pb_tlk_v6`, restype 2018) from nwsync for Spanish names/descriptions (IDs > 16777216 / 0x1000000).
- **D-05:** Extract the base game TLK from the Steam installation for standard NWN1 text (IDs < 16777216).
- **D-06:** All Name and Description fields in compiled catalogs are resolved to Spanish text. TLK IDs are not exposed to the planner.

### Catalog scope
- **D-07:** Produce compiled catalogs for ALL character-building data in one pass: classes, races, subraces, skills, feats, spells, domains, deities, and their cross-reference tables (cls_skill_*, cls_feat_*, cls_spell_*, etc.).
- **D-08:** Filter to player-relevant content only: PlayerClass=1 classes (47), PlayerRace=1 races (19), non-deprecated subraces, non-hidden skills, feats available to players, spells castable by player classes. Exclude monster/NPC/creature-only data.
- **D-09:** The extractor handles Puerta-specific customizations: renamed skills (Leadership, Sail, CraftMastery, Climb), 11 custom skills (indices 28-38), custom columns (GS_IP_DISABLED, HideFromLevelUp), shared skill tables between classes (FavoredSoul→CLS_SKILL_CLER).

### Output format
- **D-10:** Compiled catalogs are TypeScript files with typed `export const` declarations, imported statically into the planner bundle at build time. Same pattern as the current `compiled-skill-catalog.ts`.
- **D-11:** Each catalog is a separate file under `apps/planner/src/data/` (or equivalent): `compiled-classes.ts`, `compiled-skills.ts`, `compiled-feats.ts`, `compiled-spells.ts`, `compiled-domains.ts`, `compiled-races.ts`, `compiled-deities.ts`.
- **D-12:** Each compiled file includes a `datasetId` with provenance (server manifest hash + extraction timestamp) for dataset version tracking.

### Data validation
- **D-13:** Best-effort parsing with warnings. Include everything the extractor can parse. Mark inconsistencies (missing TLK entries, `****` fields, orphan references) as warnings in the extraction log, but do not exclude data.
- **D-14:** The extraction log is a human-readable report listing: total items per catalog, warnings, skipped items with reason, and dataset provenance.

<specifics>
## Specific Ideas

- The nwsync NSYC format was reverse-engineered during this session: NSYC header (4 bytes) + version uint32LE (4 bytes) + sub-header (16 bytes including compression type and uncompressed size) + zstd compressed payload at offset 24. Python `zstandard` library confirmed working.
- The base game BIF/KEY parsing was also proven during this session: KEY V1 format indexes resources by resref+restype, BIF V1 stores resources at offsets with 16-byte variable resource table entries.
- TLK custom IDs from Puerta use the offset convention: actual_id = stored_id - 16777216 (0x1000000). The `pb_tlk_v6` resource contains the server's custom string table.
- Server has 59 cls_skill_* tables, 47 player classes, and two "formats": 19 classes use an expanded 39-skill set (indices 0-38), 40 classes use the standard 28-skill set (indices 0-27).
- Skill cost rules are NWN1 standard: class skills cost 1 point per rank (cap = level+3), cross-class cost 2 points per rank (cap = floor((level+3)/2)). All ranks are integers.
- The `.planning/phases/05-skills-derived-statistics/server-extract/` directory contains 355 pre-extracted .2da files from a previous manual extraction step. These can be used as a reference/validation source.

</specifics>
