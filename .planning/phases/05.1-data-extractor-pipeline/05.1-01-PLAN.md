  <action>
1. **NSYC decompressor** (`nsyc-decompress.ts`): Per RESEARCH Pattern 1 and D-02.
   - Validate 4-byte "NSYC" magic header.
   - Read uint32LE version at offset 4; reject if not 3.
   - Skip 24-byte total header (4 magic + 4 version + 16 sub-header).
   - Decompress remaining payload with `fzstd.decompress()`.
   - Return result as `Buffer.from()`.
   - Export: `decompressNsyc(blob: Buffer): Buffer`

2. **TLK parser** (`tlk-parser.ts`): Per RESEARCH Pattern 2 and D-04/D-05.
   - Validate "TLK " magic (4 bytes) + "V3.0" version (4 bytes).
   - Read header: stringCount at offset 12, stringsDataOffset at offset 16.
   - Entry table starts at byte 20, 40 bytes per entry.
   - For each entry: flags at +0, string offset at +28, string length at +32.
   - Only read text if (flags & 1) is set.
   - Decode strings as Latin-1 (`Buffer.toString('latin1')`) -- this correctly handles Spanish accented characters (per RESEARCH Pitfall 2, A2).
   - Export: `parseTlk(data: Buffer): TlkTable` with interface `TlkTable { stringCount: number; getString(index: number): string; }`

3. **2DA parser** (`two-da-parser.ts`): Per RESEARCH Pattern 4 and D-13.
   - Skip line 1 ("2DA V2.0") and blank line 2.
   - Parse column headers from line 3 (trim + split on whitespace runs).
   - Parse data rows starting at line 4: first token is row index, remaining tokens map to columns.
   - Handle quoted values: detect opening `"`, consume until closing `"`.
   - Convert `****` to `null` (per RESEARCH anti-pattern).
   - Handle rows with fewer values than columns (trailing columns default to null, per RESEARCH Pitfall 3).
   - Handle non-contiguous row indices (use Map<number, ...>, per RESEARCH anti-pattern about gaps).
   - Export: `parseTwoDa(content: string): TwoDaTable` with interface `TwoDaTable { columns: string[]; rows: Map<number, Record<string, string | null>>; getCell(row: number, column: string): string | null; rowCount: number; }`

4. **Tests**: Write tests FIRST (RED), then implement (GREEN). Use synthetic test fixtures for unit tests:

   - TLK: Construct a minimal TLK V3.0 binary buffer with 2-3 test strings including one with Spanish accents ("Paladín").
   - 2DA: Use string literals with representative 2DA content including `****` values, quoted strings, and gaps.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| nwsync blob -> decompressor | Untrusted binary data from SQLite database |
| TLK binary -> parser | Untrusted binary with offset-based reads |
| 2DA text -> parser | Untrusted text with variable-width columns |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05.1-01 | Tampering | nsyc-decompress.ts | mitigate | Validate NSYC magic and version before decompression; fzstd throws on corrupted payloads |
| T-05.1-02 | Tampering | tlk-parser.ts | mitigate | Bounds-check entry index against stringCount; bounds-check string offset+length against buffer size before reading |
| T-05.1-03 | Tampering | two-da-parser.ts | accept | 2DA is text-parsed at build time only; malformed rows produce null cells, not crashes |
| T-05.1-04 | Information Disclosure | config.ts | accept | Machine-local paths in config.ts are build-time only; dataset-manifest.ts windowsAbsolutePathRegex prevents them from leaking into compiled output |
</threat_model>
