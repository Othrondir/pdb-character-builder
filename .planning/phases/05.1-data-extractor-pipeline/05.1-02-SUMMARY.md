## Decisions Made
- Used dependency injection (optional `readFile` parameter) instead of ESM `vi.spyOn` mocking for BaseGameReader -- ESM module namespaces are not configurable in Vitest, making spy-based mocking fail. DI is cleaner and more testable.
- Named test file `.spec.ts` instead of plan-specified `.test.ts` to match the existing vitest include pattern (`tests/**/*.spec.ts`).
