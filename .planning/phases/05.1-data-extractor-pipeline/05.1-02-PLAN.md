<!-- Binary format specs from RESEARCH.md -->
BIF V1.0: "BIFF" magic + "V1  " version. Header 20 bytes. Variable resource table: 16-byte entries (id, offset, size, type).
KEY V1.0: "KEY " magic + "V1  " version. Header fields: BIF count at 8, key count at 12, file table offset at 16, key table offset at 20. BIF file table entries: size(4) + filename_offset(4) + filename_size(2) + drives(2). Key table entries: 22 bytes (resref 16 + restype 2 + resid 4). BIF index = (resid >> 20) & 0xFFF, resource index = resid & 0xFFFFF.

  <behavior>
    BIF parser:
    - parseBif(validBifBuffer) returns a list of BifResourceEntry with offset, size, type
    - parseBif(invalidMagic) throws "Not a BIF file"
    - parseBif extracts resource data by index using offset+size from the entry table

    KEY parser:
    - parseKey(validKeyBuffer) returns KeyEntry[] with resref, restype, bifIndex, resourceIndex
    - parseKey(invalidMagic) throws "Not a KEY file"
    - parseKey correctly maps BIF file table entries to relative BIF paths
    - parseKey resolves bifIndex from (resid >> 20) & 0xFFF and resourceIndex from resid & 0xFFFFF

    BaseGameReader:
    - constructor(keyPath, baseDir) parses the KEY file and indexes BIF references
    - getResource(resref, restype) returns the raw Buffer for a matching resource or null
    - getResource for a known base-game 2DA (e.g., "subraces", 2017) returns non-null content
  </behavior>
  <action>
1. **BIF parser** (`bif-parser.ts`): Per RESEARCH BIF V1.0 format.
   - Validate "BIFF" + "V1  " magic (8 bytes).
   - Read variable resource count at offset 8, variable table offset at offset 16.
   - Parse variable resource table: each entry is 16 bytes (id u32, offset u32, size u32, type u32).
   - Export `parseBif(data: Buffer): BifFile` where `BifFile` has `getResource(index: number): Buffer | null` using offset+size to slice the BIF data.
   - Export `BifResourceEntry = { id: number; offset: number; size: number; type: number; }`.

2. **KEY parser** (`key-parser.ts`): Per RESEARCH KEY V1.0 format.
   - Validate "KEY " + "V1  " magic (8 bytes).
   - Read: bifCount at 8, keyCount at 12, fileTableOffset at 16, keyTableOffset at 20.
   - Parse BIF file table: each entry has fileSize(4), filenameOffset(4), filenameSize(2), drives(2). Read filename as ASCII from the KEY buffer at filenameOffset for filenameSize bytes.
   - Parse key table: each entry is 22 bytes. ResRef = 16 bytes null-padded ASCII, restype = uint16LE at +16, resid = uint32LE at +18. Compute bifIndex = (resid >> 20) & 0xFFF, resourceIndex = resid & 0xFFFFF.
   - Export `parseKey(data: Buffer): KeyFile` where `KeyFile` has `bifPaths: string[]` and `entries: KeyEntry[]`.
   - Export `KeyEntry = { resref: string; restype: number; bifIndex: number; resourceIndex: number; }`.

3. **BaseGameReader** (`base-game-reader.ts`): Orchestrates KEY + BIF to resolve resources.
   - Constructor takes `keyPath: string` and `baseDir: string`.
   - Reads and parses the KEY file. Indexes entries by `resref:restype` for O(1) lookup.
   - Caches parsed BIF files lazily (only parse a BIF when first resource from it is requested).
   - `getResource(resref: string, restype: number): Buffer | null`: looks up KeyEntry, gets bifIndex, loads + parses BIF if not cached, extracts resource by resourceIndex.
   - BIF file paths from KEY are relative to `baseDir` (e.g., `data/base_2da.bif`). Normalize path separators.
   - NOTE: The base game `dialog.tlk` is NOT inside a BIF -- it is a standalone file at `lang/es/data/dialog.tlk`. The reader has a separate `getTlk(path: string): Buffer` that reads the file directly (for Plan 03's TLK resolver to use). Do NOT parse dialog.tlk through BIF/KEY.

4. **Tests**: Write tests FIRST. For BIF/KEY, construct minimal valid binary buffers by hand:
   - BIF: BIFF+V1 header, 1 variable resource entry pointing to a 5-byte payload at a known offset.
   - KEY: KEY +V1 header, 1 BIF file table entry, 1 key table entry pointing to the BIF.
   - BaseGameReader: Test with a mock that returns synthetic KEY and BIF buffers (don't require actual game files in CI -- use dependency injection or a factory). Add a SKIPPED integration test that reads from the actual Steam install path if the file exists.
