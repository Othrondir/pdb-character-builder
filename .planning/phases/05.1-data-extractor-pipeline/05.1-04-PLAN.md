<objective>
Implement the three most complex catalog assemblers: feats (with prerequisite columns and class feat lists), spells (with class-level mappings and gain/known tables), and domains (with granted feats and spell lists). These are the cross-reference-heavy assemblers that tie together multiple 2DA tables.

Purpose: Feats, spells, and domains are the richest data domains in the extractor. Feats cross-reference cls_feat_* tables per class. Spells cross-reference cls_spgn_* and cls_spkn_* tables (some in nwsync, some in base game BIF). Domains reference both feat and spell indices. Separating these from the simpler assemblers keeps context cost manageable.

Output: Three assembler functions producing Zod-validated catalog payloads for feats, spells, and domains.
</objective>

  <action>
`assembleFeatCatalog(nwsyncReader, baseGameReader, tlkResolver, classRows): { catalog: FeatCatalog, warnings: string[] }`

The `classRows` parameter is a map of class canonical IDs to their `sourceRow` + `featTableRef` from the class assembler output, so the feat assembler knows which cls_feat_* tables to load and how to map them back to class IDs.

1. **Load feat.2da** from nwsync (resref "feat", restype 2017).
   - Per RESEARCH: nwsync has ~1823 lines. The pre-extracted version had 2694 -- different counts are expected.

2. **Filter player-relevant feats** (per D-08):
   - Include feats where `ALLCLASSESCANUSE` = "1" OR the feat appears in at least one cls_feat_* table with List > 0.
   - Exclude feats that are purely monster/NPC (not in any class list and not general).
   - Per D-13: Include everything parseable; log warnings for feats with missing TLK entries but do not exclude them.

3. **Extract prerequisite data** for each feat (per RESEARCH Feat Prerequisite Column Reference):
   - Map each prerequisite column to the FeatPrerequisites shape:
     - `minBab`: parseInt(MINATTACKBONUS) or null
     - `minStr`..`minCha`: parseInt of corresponding MIN* columns or null
     - `minSpellLevel`: parseInt(MINSPELLLVL) or null
     - `minLevel`: parseInt(MinLevel) or null
     - `maxLevel`: parseInt(MaxLevel) or null
     - `minLevelClass`: parseInt(MinLevelClass) or null
     - `minFortSave`: parseInt(MinFortSave) or null
     - `requiredFeat1`, `requiredFeat2`: feat index -> `feat:slug` canonical ID (or null)
     - `orReqFeats`: OrReqFeat0..OrReqFeat4 -> array of feat canonical IDs (filter nulls)
     - `requiredSkill`: REQSKILL index + ReqSkillMinRanks -> `{ skillId: 'skill:slug', minRanks: N }` or null
     - `requiredSkill2`: same for REQSKILL2 + ReqSkillMinRanks2
     - `preReqEpic`: boolean from PreReqEpic column
   - For feat cross-references (PREREQFEAT1, OrReqFeat*, REQSKILL): the values are row indices into feat.2da or skills.2da. Build a lookup map `rowIndex -> canonicalId` before processing prerequisites.

4. **Generate canonical IDs**: `feat:${LABEL.toLowerCase().replace(/_/g, '-')}`. LABEL column from feat.2da is the English internal name. If duplicates arise (unlikely but possible), append `-${sourceRow}`.

5. **Build class feat lists** (`classFeatLists`):
   - For each player class's `featTableRef` (e.g., "CLS_FEAT_FIGH"):
     - Load the cls_feat_* 2DA from nwsync (resref = featTableRef lowercased, restype 2017).
     - If not in nwsync, try base game BIF fallback.
     - Parse each row: `FeatIndex` -> feat canonical ID, `List` (0-3), `GrantedOnLevel` (parseInt or null for -1), `OnMenu` (boolean from "1"/"0").
     - Map to the class's canonical ID as the key.
   - Per RESEARCH: 44 cls_feat_* tables exist in nwsync.

6. **Warnings** (per D-13): Log warnings for feats with `****` Name (missing TLK), feats with `****` LABEL (missing internal name), orphan feat references in prerequisites.

7. Validate output with `featCatalogSchema.parse()`.

**Spell Assembler**: `assembleSpellCatalog(nwsyncReader, baseGameReader, tlkResolver, classRows): { catalog: SpellCatalog, warnings: string[] }`

1. **Load spells.2da** from nwsync (resref "spells", restype 2017).
   - Per RESEARCH: ~1558 lines in nwsync.

2. **Extract class-level mappings** for each spell:
   - Check each class spell column: Bard, Cleric, Druid, Paladin, Ranger, Wiz_Sorc, and Puerta custom columns: Shaman, FavSoul, Hex, Warlock, Spellsword, InvokerEle, InvokerHem.
   - For each column with a numeric value (not "****"), add to `classLevels` record: key = canonical class ID, value = parseInt (spell level 0-9).
   - Map column names to canonical class IDs: "Bard" -> "class:bard", "Cleric" -> "class:cleric", "Wiz_Sorc" -> "class:wizard" (or appropriate mapping from classes.2da), "FavSoul" -> "class:favored-soul", etc. The `classRows` parameter provides the column-name-to-classId mapping.
   - `innateLevel`: parseInt of "Innate" column or null.

3. **Filter to player-castable spells** (per D-08):
   - Include spells that have at least one entry in `classLevels` for a player class. Exclude spells that are monster/creature-only (no player class columns have values).
   - Per D-13: Include spells with parse warnings rather than excluding them.

4. **Generate spell IDs**: `spell:${LABEL.toLowerCase().replace(/_/g, '-')}`. Handle duplicates by appending sourceRow.

5. **Build spell gain tables** (`spellGainTables`):
   - For each spellcasting class (classes with non-null `spellGainTableRef`):
     - Load the cls_spgn_* table: first from nwsync (resref = table name lowercased), then base game BIF fallback (per RESEARCH Pitfall 6 -- only 7 tables in nwsync, rest in base game).
     - Parse each row: level, SpellLevel0..SpellLevel9 values.
     - Key by canonical class ID.

6. **Build spell known tables** (`spellKnownTables`):
   - Same pattern as spell gain. Only 4 cls_spkn tables in nwsync; rest from base game.
   - Per RESEARCH: Not all classes have spell known tables (prepared casters like Clerics and Wizards don't need them).

7. Validate output with `spellCatalogSchema.parse()`.

**Domain Assembler**: `assembleDomainCatalog(nwsyncReader, baseGameReader, tlkResolver, featIdsByRow, spellIdsByRow): { catalog: DomainCatalog, warnings: string[] }`

The `featIdsByRow` and `spellIdsByRow` parameters are maps from feat.2da/spells.2da row indices to canonical IDs, produced by the feat and spell assemblers respectively.

1. **Load domains.2da** from nwsync (resref "domains", restype 2017).
   - Per RESEARCH: 34 entries.

2. **For each domain row**:
   - `id`: `domain:${Name-column-label-slug}` or `domain:${row-index}` if no label column.
   - `label`: `tlkResolver.resolve(parseInt(Name))` -- Spanish name.
   - `description`: `tlkResolver.resolve(parseInt(Description))` -- Spanish description.
   - `grantedFeatIds`: The `GrantedFeat` column contains a feat.2da row index. Map to feat canonical ID via `featIdsByRow`. If multiple granted feats exist (some domains grant 2+), check columns GrantedFeat1, GrantedFeat2, etc. If only one GrantedFeat column, produce a single-element array.
   - `spellIds`: 10-element array (spell levels 0-9). For each `Level_0` through `Level_9` column, the value is a spells.2da row index. Map to spell canonical ID via `spellIdsByRow`. Produce null for "****" values.
   - `sourceRow`: row index.

3. Validate output with `domainCatalogSchema.parse()`.
