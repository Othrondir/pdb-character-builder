## Fixed Issues

### WR-01: Focus-management effect steals focus on initial mount

**Files modified:** `apps/planner/src/components/shell/mobile-nav-toggle.tsx`
**Commit:** b3c9e28
**Applied fix:** Introduced `wasOpenRef` ref (initialised `false`) and gated the
focus-restoration branch on `wasOpenRef.current`. The effect now distinguishes
"drawer just closed" (wasOpenRef true -> restore focus to toggle) from
"drawer is in default-closed state on mount" (wasOpenRef false -> no-op). On
open, the ref is flipped to `true` before the rAF-scheduled focus call.

### WR-02: `aria-modal="true"` asserted without a focus trap

**Files modified:** `apps/planner/src/components/shell/planner-shell-frame.tsx`
**Commit:** 1ea1cd2
**Applied fix:** Chose the "cheapest path" from the review: dropped the
`aria-modal` prop entirely. The drawer still carries `role="dialog"` while
open (structural semantics), but no longer advertises modality it does not
enforce. A real focus trap is not installed — deferred until the phase
owner chooses whether to pull in `focus-trap-react` or ship a bespoke trap.

### WR-03: No test coverage for the accessibility contract

**Files modified:**
- `tests/phase-07.1/mobile-nav-close-affordances.spec.tsx`
- `tests/phase-07.1/planner-shell-drawer.spec.tsx` (new file)

**Commit:** d8ad574
**Applied fix:**
- Added three assertions to `mobile-nav-close-affordances.spec.tsx`:
  1. Opening the drawer moves focus to the close button (flushes rAF before asserting).
  2. Closing via Escape returns focus to the toggle button.
  3. Initial mount does NOT steal focus from an outside element (WR-01 regression guard).
- Added new integration spec `planner-shell-drawer.spec.tsx` rendering the full
  `PlannerShellFrame`. Asserts:
  - Drawer has no `role` attribute when `mobileNavOpen` is false.
  - Drawer has `role="dialog"` while `mobileNavOpen` is true.
  - Drawer does NOT carry `aria-modal` (locks in the WR-02 fix).

### WR-04: Body scroll is not locked while drawer is open

**Files modified:** `apps/planner/src/components/shell/mobile-nav-toggle.tsx`
**Commit:** 66fa4d9
**Applied fix:** Added a third `useEffect` that toggles `document.body.style.overflow`
to `'hidden'` while `mobileNavOpen === true`, preserving and restoring the
previous inline overflow value in the cleanup callback. Bail-outs handle the
`mobileNavOpen === false` and `typeof document === 'undefined'` (SSR) cases.
