<objective>
Give the Cargar (load-slot) path the same D-07 / SHAR-05 fail-closed parity that `share-entry.tsx` already implements for URL-share and JSON-import paths. Today, `LoadSlotDialog.onPick` goes straight from `loadSlot` → `hydrateBuildDocument` with no ruleset/dataset version check, so a slot saved under an older ruleset can silently hydrate into the current planner and desync validation.

Fix: call `diffRuleset(doc)` after `loadSlot` returns and before `hydrateBuildDocument`. If `diff !== null`, render `VersionMismatchDialog` offering Descargar JSON + Cancelar terminal actions (same two terminals as share-entry). Neither terminal hydrates.

Purpose: Close SHAR-02 (local save parity) and SHAR-05 (slot-load fail-closed) by reusing the single mismatch UI surface introduced in Phase 8. No new dialog component, no new persistence helper — all pieces already exist.
Output: Updated `save-slot-dialog.tsx` LoadSlotDialog with the gate; one regression test covering both branches.
</objective>
