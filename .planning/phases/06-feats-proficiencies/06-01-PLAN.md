**1. `bab-calculator.ts`** -- Pure functions to compute BAB and saving throws from class progression.

```typescript
import type { ClassCatalog } from '@data-extractor/contracts/class-catalog';
import type { CanonicalId } from '../contracts/canonical-id';
```

Export `computeTotalBab(classLevels: Record<string, number>, classCatalog: ClassCatalog): number`:
- For each class in `classLevels`, find the class in `classCatalog.classes` by `id`.
- Read `attackBonusProgression` ('low' | 'medium' | 'high').
- Compute per-class BAB: `high` = classLevel, `medium` = `Math.floor(classLevel * 3 / 4)`, `low` = `Math.floor(classLevel / 2)`.
- CRITICAL per Pitfall 3: Floor each class contribution independently BEFORE summing. BAB = sum of floored per-class values.
- If a classId is not found in the catalog, skip it (contribute 0).

Export `computeFortSave(classLevels: Record<string, number>, classCatalog: ClassCatalog): number`:
- For each class, read `savingThrows.fortitude` ('low' | 'high').
- `high` = `2 + Math.floor(classLevel / 2)`, `low` = `Math.floor(classLevel / 3)`.
- Sum all per-class contributions (D&D 3.5 stacking).

Export `computeRefSave(classLevels, classCatalog): number` and `computeWillSave(classLevels, classCatalog): number` following the same pattern using `savingThrows.reflex` and `savingThrows.will`.

```typescript
export interface BuildStateAtLevel {

  classLevels: Record<string, number>;    // classId -> number of levels in that class at this point
```

Export interface `PrerequisiteCheck`:
```typescript
export interface PrerequisiteCheck {
  type: 'ability' | 'bab' | 'feat' | 'skill' | 'level' | 'class-level' | 'spell-level' | 'fort-save' | 'or-feats' | 'max-level' | 'epic';
  label: string;       // Spanish display label
```

Export interface `PrerequisiteCheckResult`:
```typescript
export interface PrerequisiteCheckResult {
  met: boolean;          // true if ALL checks pass
  checks: PrerequisiteCheck[];
}
```

Export `evaluateFeatPrerequisites(feat: CompiledFeat, buildState: BuildStateAtLevel, featCatalog: FeatCatalog): PrerequisiteCheckResult`:

Check each prerequisite field on `feat.prerequisites` in order. For each non-null/non-undefined field, push a `PrerequisiteCheck`:

- `minStr/minDex/minCon/minInt/minWis/minCha`: type='ability', label = Spanish name (Fuerza/Destreza/Constitucion/Inteligencia/Sabiduria/Carisma), met = `buildState.abilityScores[key] >= value`, required = `String(value)`, current = `String(buildState.abilityScores[key])`
- `minBab`: type='bab', label='BAB', met = `buildState.bab >= value`, required = `+${value}`, current = `+${buildState.bab}`
- `requiredFeat1`: type='feat', label = lookup `featCatalog.feats.find(f => f.id === id)?.label ?? id`, met = `buildState.selectedFeatIds.has(id)`, current = met ? '(tomada)' : '(no tomada)'
- `requiredFeat2`: same as requiredFeat1
- `orReqFeats`: type='or-feats', label = join labels of all feats with ' o ', met = `orReqFeats.some(id => buildState.selectedFeatIds.has(id))`, current = met ? '(cumplido)' : '(ninguna tomada)'
- `requiredSkill` / `requiredSkill2`: type='skill', label = lookup skill label from id or use id, met = `(buildState.skillRanks[id] ?? 0) >= minRanks`, required = `${minRanks} rangos`, current = `${buildState.skillRanks[id] ?? 0} rangos`
- `minLevel`: type='level', label='Nivel de personaje', met = `buildState.characterLevel >= value`, required = `String(value)`, current = `String(buildState.characterLevel)`
- `minLevelClass`: type='class-level'. This is a classId string. The required level comes from `minLevel` if present, or defaults to checking that the class exists in `buildState.classLevels`. Search for matching class in catalog for label. met = `(buildState.classLevels[minLevelClass] ?? 0) >= (feat.prerequisites.minLevel ?? 1)`.
- `minSpellLevel`: type='spell-level', label='Nivel de conjuro', met = `buildState.spellcastingLevel >= value`. For Phase 6, spellcastingLevel will be 0, so spellcaster feats will correctly show as unmet. required = `String(value)`, current = `String(buildState.spellcastingLevel)`
- `minFortSave`: type='fort-save', label='Salvacion de Fortaleza', met = `buildState.fortitudeSave >= value`, required = `+${value}`, current = `+${buildState.fortitudeSave}`
- `preReqEpic` (if true): type='epic', label='Nivel epico', met = false (epic level is 21+, planner caps at 16), required = 'Nivel 21+', current = `Nivel ${buildState.characterLevel}`
- `maxLevel` (if present): type='max-level', label='Nivel maximo', met = `buildState.characterLevel <= value`, required = `<= ${value}`, current = `String(buildState.characterLevel)`

Return `{ met: checks.every(c => c.met), checks }`.

Export `determineFeatSlots(characterLevel: number, classId: CanonicalId | null, classLevelInClass: number, classFeatLists: FeatCatalog['classFeatLists']): FeatSlotsAtLevel`:

- `autoGrantedFeatIds`: collect all entries from `classFeatLists[classId]` where `list === 3 && grantedOnLevel === classLevelInClass`. Also collect entries where `(list === 1 || list === 2) && grantedOnLevel === classLevelInClass && onMenu === false` (auto-granted silently).
- `generalFeatSlot`: true if `characterLevel` is in `[1, 3, 6, 9, 12, 15]`. NOTE: Human bonus feat at level 1 is not modeled here (would need race data); leave as a TODO comment.
- `classBonusFeatSlot`: true if any entry in `classFeatLists[classId]` has `(list === 1 || list === 2) && grantedOnLevel === classLevelInClass && onMenu === true`. This means the class grants a bonus feat pick at this class level.

Export type `FeatSlotType = 'class-bonus' | 'general'`;

Export interface `EligibleFeatSet`:
```typescript
export interface EligibleFeatSet {
  classBonusFeats: CompiledFeat[];    // feats eligible for the class bonus slot
  generalFeats: CompiledFeat[];       // feats eligible for the general slot
}
```

Export `getEligibleFeats(buildState: BuildStateAtLevel, classId: CanonicalId | null, classLevelInClass: number, featCatalog: FeatCatalog, classCatalog: ClassCatalog): EligibleFeatSet`:

- Start with all feats from `featCatalog.feats`.
- Exclude any feat where `id` is already in `buildState.selectedFeatIds` (per Pitfall 6: feats are one-time selections).
- Exclude any feat where `preReqEpic === true` (epic feats unreachable at level 1-16).
- For each remaining feat, call `evaluateFeatPrerequisites(feat, buildState, featCatalog)`. Keep only feats where `result.met === true` (per D-01: only eligible feats shown by default).
- Split into two sets:
  - `classBonusFeats`: feats where `classId` exists and `classFeatLists[classId]` has an entry with `featId === feat.id && (list === 1 || list === 2)`.
  - `generalFeats`: all eligible feats (those with `allClassesCanUse === true` OR present in classFeatLists with `list === 0`). Per D-02, general feats include everything available at the general feat level.

Export `evaluateAllFeatsForSearch(buildState: BuildStateAtLevel, featCatalog: FeatCatalog): Array<{ feat: CompiledFeat; prereqResult: PrerequisiteCheckResult }>`:
- For each feat in `featCatalog.feats`, evaluate prerequisites and return the feat + result. Used by the search feature (D-06) to show both eligible and blocked feats with reasons.
