**2. [Rule 3 - Blocking / tool-mechanics] PowerShell-based line-range deletion replaced Edit-tool old_string approach**
- **Found during:** Task 1 GREEN step (compiled-races.ts edit).
- **Issue:** `Edit(old_string=<16-line CRLF block>)` failed with "String to replace not found in file" despite the exact text existing — Windows CRLF line endings confused the tool's normalization on long multi-line matches. - **Fix:** Ran `git checkout -- apps/planner/src/data/compiled-races.ts` to restore clean HEAD state, then used `powershell -Command 'ReadAllLines + RemoveRange(200, 16) + WriteAllText with CRLF'` for precise CRLF-safe line-range excision. Same programmatic approach reused for the puerta-point-buy.json + .md deletions.
- **Files modified:** None beyond the planned set — the `git checkout` reverted only the corrupt in-progress edit; no production data lost.
- **Verification:** Post-edit grep + Read confirmed the file had exactly 732 lines (748 − 16), `race:halfelf2` count = 0, adjacent `race:drow` → `race:enano-dorado` block boundary clean.
