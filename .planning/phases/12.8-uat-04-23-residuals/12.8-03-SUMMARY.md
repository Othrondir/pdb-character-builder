## Issues Encountered

- **Vitest default esbuild + JSX:** This repo does not configure the automatic JSX runtime for Vitest (it is configured for the Vite dev build via `@vitejs/plugin-react`, but Vitest at repo root runs its own esbuild transform). Any new `.spec.tsx` that uses JSX will fail with `React is not defined`. Use `createElement(Component, props)` per the phase-12.x convention.
- **RTL multi-`it` state leakage:** `screen.getByRole` queries the entire DOM, and without explicit `cleanup` between tests the previous render's nodes remain. Every Vitest RTL suite needs `afterEach(cleanup)` (or an equivalent global setup).
- **Playwright web server reuse:** The fixture helper uses `reuseExistingServer: true`, so Playwright locks onto whatever dev server is already running on `localhost:5173`. Convenient when the user already has `pnpm dev` up; harmless otherwise (Playwright spawns its own).
