## Background

Agent-driven UAT on master `809ff11` (2026-04-23) against `localhost:5173`, fixture Elfo + Neutral puro + Atributos default (FUE 8 DES 10 CON 6 INT 8 SAB 8 CAR 8) + Guerrero L1, surfaced 6 findings across 3 feature areas:

- **Habilidades scroll (F1+F2, major)**: `apps/planner/src/styles/app.css:469-478` declares `scroll-snap-type: y proximity` on `.skill-board .selection-screen__content` with `scroll-snap-align: start` on `.skill-board .skill-sheet__row`. On first entry the browser snaps to the first `.skill-sheet__row` at offset 206, clipping the sheet header. On every `+`/`-` click React reconciles, snap re-evaluates, producing visible jitter. Phase 12.7-03 R3 attempted a scroll reset via `useLayoutEffect` in `apps/planner/src/features/skills/skill-sheet.tsx:151-156`, but `scrollerRef` attaches to `.skill-sheet` (line 160) which is not scrollable (`clientHeight === scrollHeight === 1982`); the real overflow owner is `.selection-screen__content`. jsdom missed the regression because it has no layout engine / no snap.
- **Dotes multi-slot UX (F3+F4, major)**: `apps/planner/src/features/feats/feat-sheet.tsx` renders class-bonus + general sections simultaneously with two handlers (`handleSelectClassFeat`, `handleSelectGeneralFeat`). After the class slot fills, the "Ahora" pill switches to the general section but the viewport stays on the class section; the user's second click lands on class rows again and overwrites the first pick. Once both slots fill, `<FeatSummaryCard>` (`apps/planner/src/features/feats/feat-summary-card.tsx`, Phase 12.4-07) replaces the sheet with chips + a single "Modificar selección" button; chips are not deselectable in place.
- **Prestige gate coverage (F5, minor)**: At L2 Elfo Guerrero (BAB=2), `apps/planner/src/features/level-progression/prestige-prereq-data.ts` (+ fail-closed branch in `packages/rules-engine/src/progression/prestige-gate.ts`) gates 18/21 prestige classes with real prereqs, but `pale-master` + `caballero-arcano` render OPEN (partial enrichment skips the fail-closed branch), and `shadowdancer` surfaces only BAB (feat + skill-rank prereqs missing from extractor output).
- **Race roster dedupe (F6, minor)**: `apps/planner/src/features/character-foundation/foundation-fixture.ts` dedupe (`dedupeByCanonicalId`, Phase 12.1-02) closes `race:drow` duplicate but a second `Semielfo` row escapes via a different canonical-id path, yielding 45 race picker rows (should be 44).

## Requirements

1. **F1 Habilidades scroll reset**: On first entry to the Habilidades sub-step for a level, the real scroll container is at `scrollTop === 0`.
   - Current: `.selection-screen__content.scrollTop = 206` on first entry (UAT evidence). `scrollerRef` in `skill-sheet.tsx:151-156` targets `.skill-sheet` which is not the overflow owner; `scroll-snap-type: y proximity` at `app.css:473` re-snaps past any programmatic reset.
   - Target: `document.querySelector('.skill-board .selection-screen__content').scrollTop` reads `0` on the first render of Habilidades for each `activeSheet.level`, and remains `0` until the user scrolls.
   - Acceptance: Playwright scenario (Elfo + Guerrero L1 fixture) opens Habilidades and asserts `scrollTop === 0` on the element matching `.skill-board .selection-screen__content`; switching L1 → L2 via the level rail keeps `scrollTop === 0` on re-entry.

2. **F2 Habilidades no-jitter on rank change**: Clicking `+` or `-` on any skill row does not visibly shift the skill list.
   - Current: Rows keep their absolute position, but snap re-binding on React commit produces perceived jitter (UAT evidence — rows top=78 height=42 before and after click, scrollTop=206 both sides).
   - Target: Before-and-after `+`/`-` click, the bounding client rect of the clicked row and the `scrollTop` of the real scroller both match within 1px.
   - Acceptance: Playwright scenario captures `getBoundingClientRect()` on a sampled row + `scrollTop` of `.selection-screen__content` before click, performs `+` then `-`, asserts both values match exactly (no tolerance) or within a 1px fuzz.

3. **F3 Dotes active-section surfacing**: After the class-bonus dote slot fills, the general-slot section becomes the next interaction target in the user's viewport without requiring manual scroll.
   - Current: `feat-sheet.tsx` renders both sections; pill state + `feat-sheet__group--current` class swap the "Ahora" highlight but the viewport stays on the class section. Second click re-selects within class and overwrites (`counter` stays at 1/2 with the first pick replaced).
   - Target: After `handleSelectClassFeat` completes the class slot, the first interactive row of `[data-slot-section="general"]` is inside the visible viewport region without the user scrolling (implementation choice — auto-scroll vs collapse-on-complete vs viewport-nudge banner — deferred to discuss-phase).
   - Acceptance: Playwright scenario (Elfo Guerrero L1, click "Carrera" in class section) asserts (a) the first row of `[data-slot-section="general"]` has `getBoundingClientRect().top >= 0 && bottom <= viewport.height` immediately after the class-bonus selection completes, and (b) clicking any general-section row fills the general slot without mutating the class-bonus pick (`counter 2/2`, both chips distinct).

4. **F4 FeatSummaryCard deselect**: After both per-level dote slots fill, the user can clear either slot from the summary UI without re-opening the full sheet.
   - Current: `<FeatSummaryCard>` renders chips + "Modificar selección"; chips are display-only (`feat-summary-card.tsx`). `clearClassFeat` / `clearGeneralFeat` exist in feat store but are unreachable from the summary view.
   - Target: Each chip in `<FeatSummaryCard>` exposes a user-operable deselect control (button, × glyph, or toggle-chip) that invokes the matching `clearClassFeat` / `clearGeneralFeat` action. Invoking it returns the sheet to the pre-completion rendering mode for the cleared slot.
   - Acceptance: Playwright scenario fills both slots, locates the class-bonus chip in `<FeatSummaryCard>`, invokes its deselect control, asserts `<FeatSheet>` re-renders with class slot empty + general slot preserved; Vitest coverage for the chip deselect click handler is green.

5. **F5 Prestige gate fail-closed tightening + 3-class enrichment**: At L2+, no prestige class renders without a real prereq gate when its extractor enrichment is incomplete, and `pale-master`, `caballero-arcano`, and `shadowdancer` surface server-accurate prereqs.
   - Current: `prestige-prereq-data.ts` + `prestige-gate.ts` render OPEN for `pale-master` (real reqs: non-good align + Hechizar 4 + Saber religion 2 + Toughness) and `caballero-arcano` (real reqs: BAB + spellcasting); `shadowdancer` surfaces only `Requiere BAB ≥ 2` (real reqs: Move Silent 8 + Tumble 5 + Dodge + Mobility).
   - Target: Fail-closed branch detects partial enrichment (via explicit per-class flag, heuristic on prereq-list shape, or source-of-truth override table) and renders `'Requisitos en revisión'` rather than OPEN; `pale-master`, `caballero-arcano`, `shadowdancer` ship overrides matching Puerta server reqs (pattern: `.planning/quick/260422-h9k-...` extended).
   - Acceptance: Vitest snapshot at L2 Elfo Guerrero BAB=2 asserts (a) no prestige row has `disabled=false` without a prereq label, (b) `pale-master` / `caballero-arcano` render the fail-closed sentinel OR a real prereq label, (c) `shadowdancer` label includes feat + skill-rank terms (not BAB-only).

6. **F6 Semielfo roster dedupe**: The race picker renders `Semielfo` exactly once.
   - Current: 45 rows, duplicate label `Semielfo` (UAT evidence). `dedupeByCanonicalId` in `foundation-fixture.ts` closes `race:drow` duplicate but a second semielfo emits under a different canonical-id path.
   - Target: Race picker renders ≤ 44 rows and the `Semielfo` label appears exactly once.
   - Acceptance: Vitest / Playwright assertion on the race picker queries all `[data-race-label="Semielfo"]` (or equivalent stable selector) and asserts `.length === 1`.
