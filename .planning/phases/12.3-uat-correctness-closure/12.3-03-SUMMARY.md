# Phase 12.3 Plan 03: Per-Level Dotes Gate + Slot Prompt Summary

Scopes the Dotes sub-step empty-state to the ACTIVE level's classId instead of a GLOBAL `any-level-has-a-class` check and surfaces a Spanish slot-availability prompt above the feat sheet — closing UAT B3 (CRITICAL false-block) and B4 (HIGH missing slot count) from UAT-FINDINGS-2026-04-18.md.

## What Shipped

### 1. Per-level empty-state gate (UAT B3)

`selectFeatBoardView` previously computed `progressionHasClass = progressionState.levels.some(l => l.classId !== null)` and returned the empty sheet with `emptyStateBody: shellCopyEs.feats.emptyStateBody` when that global was false. The gate is now active-level-scoped:

```ts
const activeLevel = featState.activeLevel;
const activeProgression = progressionState.levels.find((r) => r.level === activeLevel) ?? null;
const classId = activeProgression?.classId ?? null;

if (classId === null) {
  // empty sheet + emptyStateBody: shellCopyEs.feats.emptyStateBodyPerLevel
}
```

With 12.3-02's active-level wiring, clicking any rail level makes `featState.activeLevel === N`, so the gate reflects the level the user is looking at. The new Spanish copy (`emptyStateBodyPerLevel: 'Selecciona una clase para este nivel antes de asignar dotes.'`) points users to the local action instead of the confusing global "Completa una progresion valida en Construccion" message.

### 2. slotPrompt field (UAT B4)

Added `ActiveFeatSheetView.slotPrompt: string | null`. A pure helper composes it from `determineFeatSlots` output plus current selections:

```ts
function computeSlotPrompt(
  featSlots: { classBonusFeatSlot: boolean; generalFeatSlot: boolean },
  selectedClassFeatId: CanonicalId | null,
  selectedGeneralFeatId: CanonicalId | null,
): string | null {
  const parts: string[] = [];
  if (featSlots.classBonusFeatSlot && selectedClassFeatId === null) {
    parts.push(shellCopyEs.feats.slotPromptClassAvailable);
  }
  if (featSlots.generalFeatSlot && selectedGeneralFeatId === null) {
    parts.push(shellCopyEs.feats.slotPromptGeneralAvailable);
  }
  return parts.length > 0 ? parts.join(' ') : null;
}
```

Shape examples:
- L1 Guerrero empty → `'Dote de clase disponible. Dote general disponible.'`
- L1 Guerrero + class-bonus feat picked → `'Dote general disponible.'`
- L1 Guerrero fully filled → `null`
- L2 empty (no class) → `null` (empty-state branch short-circuits)
- L2 Pícaro with no slots this level → `null`

### 3. FeatBoard render

FeatBoard renders the prompt above the FeatSheet when truthy:

```tsx
<SelectionScreen title={title} className="feat-board">
  {boardView.activeSheet.slotPrompt ? (
    <p className="feat-board__slot-prompt">{boardView.activeSheet.slotPrompt}</p>
  ) : null}
  <FeatSheet ... />
  <FeatDetailPanel ... />
</SelectionScreen>
```
