## Decisions Made

- **Gap-before-first-configured-level uses L2+ formula** (ordinal position, not "first-configured-semantic"). Locked by Suite D test: `L1=null + L2=Guerrero + CON 0 → 6`, not 10. Consistent with how `progression-revalidation` treats inheritance from upstream configured slots. Not the only defensible reading — preserves an invariant that identical ordinal positions always use the same formula regardless of upstream gaps.
- **Per-level floor of 1 inside the selector** (not at render layer). Keeps the NWN minimum pure and reusable by any future HP consumer (e.g. a magic scaling feat, a constitution-damage debuff preview) without reintroducing the rule.
- **Empty progression returns 0; render layer maps 0 → `--`**. Preserves the legacy "no class configured" UX signal while keeping the selector's output numeric and unambiguous.
- **Selector reads `hitDie` directly from the live `compiledClassCatalog`**, not from an intermediate rules abstraction. ClassCatalog is the source of truth (extractor-generated), so adding a layer would only add drift risk.
- **`.spec.ts` file with `// @vitest-environment jsdom` pragma + `createElement`** (not `.spec.tsx`). Plan specified the `.ts` extension and the pragma cleanly opts-in jsdom for this single file without touching vitest.config.ts `environmentMatchGlobs`.
