NWN1 HP formula (per CONTEXT D-04 + NWN1 server convention, canonical reference: Bioware NWN manual + Beamdog EE patch notes):
- L1: hitDie + conMod (max roll; L1 PCs always start with max HP).
- L2+: floor(hitDie / 2) + 1 + conMod (average roll rounded up — Puerta de Baldur standard).
- Each level's class's hit die applies; multiclass builds sum per-level HP contributions.
- CON mod can be negative; HP at any level floors at 1 (NWN1 minimum) per level — but the total isn't floored if the character is still under training (not a concern at L1+; verify at edge of very low CON).

Safety guard: if CON mod is extremely negative (e.g., CON 3 → -4), a fighter L1 could still show 10-4=6 HP. If CON is even lower, HP could go negative; floor each level's contribution at 1 to match NWN convention. Executor: implement `Math.max(1, hitDie + conMod)` at L1 and `Math.max(1, floor(hitDie/2) + 1 + conMod)` at L2+ — matches server behavior and prevents nonsense states.

Empty-progression handling: when all `classLevels[*].classId === null`, compute returns 0. The render layer in character-sheet maps 0 to `--` (to preserve the existing UX signal that no class means no HP to show).

    ```typescript
    import type { ClassCatalog } from '@data-extractor/contracts/class-catalog';

    /**
     * Phase 12.3-04 (D-04) — pure NWN1 hit-point selector.
     *
     * Formula (Puerta de Baldur server convention, matching NWN1 Aurora):
     *   L1 (first configured level): hitDie + conModifier (max roll).
     *   L2+ (every subsequent level of ANY class): floor(hitDie / 2) + 1 + conModifier
     *     (average roll rounded up — server rule, no random per-level rolls).
     *   Each level uses its OWN class's hit die — multiclass-aware.
     *   Each level's HP contribution floors at 1 (NWN minimum; prevents negative HP with very low CON).
     *
     * Empty / gapped progression: unconfigured levels (classId === null) contribute 0.
     * A gapped progression where L1 is null but L2 is set treats L2 as the first
     * configured level; its HP applies the L2+ formula (not the L1 max-roll) —
     * consistent with how progression-revalidation inherits upstream classes.
     *
     * Framework-agnostic. No React, no store reads, no browser APIs.
     */
    interface ProgressionLevelRecordLike {
      classId: string | null;
      level: number;
    }

    export function computeHitPoints(
      levels: ProgressionLevelRecordLike[],
      classCatalog: ClassCatalog,
      conModifier: number,
    ): number {
      let total = 0;
      let seenFirstConfigured = false;

      const sorted = [...levels].sort((a, b) => a.level - b.level);

      for (const record of sorted) {
        if (record.classId === null) continue;
        const classEntry = classCatalog.classes.find((c) => c.id === record.classId);
        if (!classEntry) continue;
        const hitDie = classEntry.hitDie;
        const contribution = seenFirstConfigured
          ? Math.floor(hitDie / 2) + 1 + conModifier
          : hitDie + conModifier;
        total += Math.max(1, contribution);
        seenFirstConfigured = true;
      }

      return total;
    }
    ```
