<decisions>
## Implementation Decisions

### A1 тАФ Snapshot file shape + loader

- **D-01:** Snapshot file path: `packages/rules-engine/src/foundation/data/puerta-point-buy.json`. Kebab-case filename consistent with existing foundation modules.
- **D-02:** Zod schema module: `packages/rules-engine/src/foundation/point-buy-snapshot.ts`. `pointBuyCurveSchema` = `z.object({ budget: z.number().int().positive(), minimum: z.number().int(), maximum: z.number().int(), costByScore: z.record(z.string().regex(/^\d+$/), z.number().int().nonnegative()) })`. `pointBuySnapshotSchema` = `z.record(canonicalRaceIdSchema, pointBuyCurveSchema)`. Parse fails at module load тЖТ fail-closed at startup, not runtime.
- **D-03:** Loader pattern: pure function `loadPointBuySnapshot(): Record<CanonicalId, AbilityBudgetRules>`. Imports JSON via `import snapshotJson from './data/puerta-point-buy.json'` (Vite/Vitest resolve). Zero React, zero filesystem reads at runtime. Exported as `PUERTA_POINT_BUY_SNAPSHOT` for selector consumption.
- **D-04:** Provenance: sibling `packages/rules-engine/src/foundation/data/puerta-point-buy.md` records the Puerta server source per race entry. Un-sourced entries block code review (not a runtime gate). No `verified` field in the JSON schema тАФ keeps runtime contract tight; provenance lives in the .md doc.

### A1 тАФ Missing-curve fail-closed routing

- **D-05:** Detection routed through the rules-engine helper: `calculateAbilityBudgetSnapshot` input signature extended from `attributeRules: AbilityBudgetRules` тЖТ `attributeRules: AbilityBudgetRules | null`. Null тЖТ returns `status: 'blocked'`, `remainingPoints: 0`, `spentPoints: 0`, issues include a new `resolveValidationOutcome({ affectedIds: ['rule:point-buy-missing'], ... })` outcome. Preserves framework-agnostic contract; UI passes through.
- **D-06:** Selector `selectAbilityBudgetRulesForRace(raceId: CanonicalId | null): AbilityBudgetRules | null` lives at `apps/planner/src/features/character-foundation/selectors.ts`. Returns `PUERTA_POINT_BUY_SNAPSHOT[raceId]` or `null`. AttributesBoard threads selector output directly into `calculateAbilityBudgetSnapshot` тАФ no UI-layer short-circuit.
- **D-07:** Copy: add `shellCopyEs.attributes.pointBuyMissing` (function variant: `(raceLabel: string) => "Curva punto-compra no disponible para ${raceLabel}"`). AttributesBoard renders the copy when `attributeRules === null`; `+` and `-` buttons carry `disabled` attribute; budget line hidden.

### A1 тАФ Per-race cost resolution

- **D-08:** `nextIncrementCost` + `canIncrementAttribute` signatures unchanged (already parametrized by `costByScore` + `maximum`). Call sites swap the uniform `attributeRules.costByScore` for the per-race curve resolved via the selector.
- **D-09:** `foundation-fixture.ts:attributeRules` (lines 205-223) removed from the RUNTIME path. May remain as a test-only fixture export for pre-Phase-12.6 suite compatibility, but the AttributesBoard no longer reads it. Runtime source of truth = snapshot.

### P5 тАФ Row component layout

- **D-10:** New component: `apps/planner/src/features/level-progression/level-progression-row.tsx`. Renders `<li data-level-row data-level="{N}" data-legality="{legal|incomplete|invalid|locked}">`. Header = `<button>` (full row clickable) with: level number + `<span data-pill="class">`, `<span data-pill="feats">`, `<span data-pill="skills">`, `<span data-pill="legality">`.
- **D-11:** Expanded-row section: conditionally rendered when `progressionState.activeLevel === N`. Hosts existing `<ClassPicker />` + requirement rows + gains + `<AbilityIncreaseControl />` + `<LevelEditorActionBar />` (from 12.4-09). Non-active rows render only the header.
- **D-12:** Pills use Unicode glyphs for legality (тЬУ тЪа тЬЧ ЁЯФТ) тАФ no new icon library. Pill CSS under new `.level-row__pill--{class|feats|skills|legality}` namespace. Row CSS `.level-progression-row`, expanded variant `.level-progression-row--expanded`.
- **D-13:** Container: `<ol className="level-progression__list">` replaces the existing `LevelRail` sidebar. CSS grid single-column, `grid-template-rows: repeat(20, auto)`. At 1280├Ч800 compact row тЙд 32px тЖТ 20 ├Ч 32 тЙИ 640px fits without scroll; expanded row adds ~400-500px (container scrolls internally only when active row is expanded).

### P5 тАФ Expansion mechanism

- **D-14:** Click handler on row `<button>` dispatches `setActiveLevel(N)` + `setExpandedLevel(N)` atomically (same pattern as 12.4-09 `LevelEditorActionBar`). No local expand state тАФ derived from `progressionState.activeLevel === level`. `aria-expanded="true|false"` on the button.
- **D-15:** `<LevelEditorActionBar />` retained inside the expanded row тАФ its `setActiveLevelSubStep('class') тЖТ setActiveLevel(N+1) тЖТ setExpandedLevel(N+1)` dispatch continues to drive level advance. Sub-step-first dispatch order (12.4-09 contract) preserved.

### P5 тАФ level-rail fate

- **D-16:** DELETE `apps/planner/src/components/shell/level-rail.tsx` and all imports. The compact row list IS the new scan surface тАФ rail was a scan proxy and becomes redundant. Migrate any remaining test fixtures querying `level-rail__button` to `[data-level-row]`. `[data-testid="advance-to-level-{N+1}"]` stays on the advance button inside the expanded row's LevelEditorActionBar (12.4-09 selector preserved for E2E).
- **D-17:** Legality classification: derive `data-legality` via a new pure selector `selectLevelLegality(progressionState, foundationState, level): 'legal' | 'incomplete' | 'invalid' | 'locked'`. Reuses existing `selectLevelCompletionState` + G1 sequential-gate predicate (already landed in hot-fix). `locked` wins when prior level incomplete; then `invalid` (has blocked issues) > `incomplete` (missing choices) > `legal`.

### Test strategy

- **D-18:** Three new specs under `tests/phase-12.6/`:
  - `point-buy-snapshot-coverage.spec.ts` тАФ build-time Zod parse + race-coverage assertion. Fails deploy on missing race.
  - `ability-budget-per-race.spec.ts` тАФ table-driven, iterates `dedupeByCanonicalId(compiledRaceCatalog.races)`; asserts canonical remaining-points at baseline + bump pattern.
  - `level-progression-scan.spec.tsx` тАФ RTL-jsdom; 1280├Ч800 viewport; asserts 20 rows + scroll equality + pill presence + expansion mechanics + legality transitions.
- **D-19:** Pre-Phase-12.6 suites (phase-03 foundation-validation, phase-12.3 attributes-budget-gate) keep passing against the uniform fixture тАФ no breaking changes to `ability-budget.ts` inputs aside from the null branch (additive).
