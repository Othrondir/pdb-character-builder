### Decisions (documented in frontmatter)

- **Removed C1 + D2 from `l1-neutral-substeps.spec.tsx` instead of migrating.** The tests asserted the `.level-rail__abbrev` label on the L1 rail tile — both before class pick (blank) and after (non-empty). Since the rail is deleted from CreationStepper entirely, no equivalent DOM exists under that harness. - **Kept Suite C of `multiclass-active-level.spec.tsx` as `describe.skip`.** Suite C was already skipped by Plan 03 (the Selecciona la clase del nivel N heading changed to static `Progresión 1-20` per Plan 01). The test assertions would need an API change (`screen.getByText` + aria-label hunt) that goes beyond the scope of a selector migration. Suite D of the same file (store-level `selectActiveLevelSheet` follows `activeLevel`) is unskipped and green — it covers the underlying contract Suite C wrapped at the UI layer.

- **Preserved `selectLevelRail` + `LevelRailEntryView` in selectors.ts.** Despite the name, these are the framework-agnostic legality oracle used by `selectLevelLegality` and `selectProgressionSummary`. Deleting the COMPONENT does not obsolete the SELECTOR. Renaming would propagate churn beyond this plan scope without behavioral change.
