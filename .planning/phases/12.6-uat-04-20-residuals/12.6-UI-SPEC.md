## NWN1 Framing

The P5 scan surface does **not** wrap the list in `<NwnFrame>`. Rationale:
- At 20 rows × 32px, a full NwnFrame gold border + inner panel-edge rectangle would compete visually with each row's own border. Nested frames break scan density.
- Existing precedent: `level-rail` (deleted) also did not wrap individual entries in NwnFrame — only the parent `.creation-stepper` surface carried framing.
- The expanded-row slot (`.level-progression-row__expanded`) carries a single 1px `--color-selection-border` edge that visually continues from the active row's header border — this is the equivalent of a light "frame" without the double-border cost.

Existing NwnFrame consumers inside the expanded slot (e.g. `SelectionScreen` inside `<ClassPicker>`) render normally — framing is preserved at the child-component level.

A1 AttributesBoard continues to render inside `<SelectionScreen>` which already provides the NwnFrame-equivalent surface; no change.

## Accessibility

**Keyboard:**
- Row header: `<button>` element, `Tab`-reachable, Enter/Space fire click handler → dispatches `setActiveLevel(N) + setExpandedLevel(N)`.
- Locked rows: `disabled={true}` native attribute blocks keyboard activation.
- ClassPicker + LevelEditorActionBar inside expanded row: existing keyboard contracts preserved (12.4-06 / 12.4-09).
- Focus management on expand: focus stays on the row header (no auto-focus jump into ClassPicker). Rationale: avoid confusing focus loss when the user re-collapses by clicking another row.

**Screen reader:**
- Row header `aria-label={shellCopyEs.progression.rowAriaTemplate.replace('{N}', level).replace('{classLabel}', classLabel ?? 'sin clase').replace('{legalityLabel}', legalityLabels[legality])}`.
- Locked row `aria-label={shellCopyEs.progression.lockedRowAriaTemplate.replace('{N}', level)}` + `aria-disabled="true"`.
- Each pill except legality: decorative glyph is visually sufficient, copy is self-describing ("Guerrero", "2/3 dotes"), no extra `aria-label`.
- Legality pill: `aria-label={legalityLabels[legality]}` — glyph alone is ambiguous to AT.
- `aria-expanded="true|false"` on row header signals expansion state.

**Color + contrast:**
- `--color-text` (#e8d5a0) on `--color-surface` (#0a0a0a): contrast ratio ≈ 13:1 → AAA.
- `--color-text-muted` (#8b7355) on `--color-surface`: ratio ≈ 5.4:1 → AA for body, AAA for 18pt+.
- Locked rows at opacity 0.45: effective contrast of text ≈ 5.9:1 (#e8d5a0 × 0.45 over #0a0a0a) → still AA for 14px body. Glyph on 14px sits at AA boundary; accepted trade-off for density (matches existing `.level-rail__button.is-locked` precedent).
- Destructive callout (`rgba(122,31,31,0.22)` background with `--color-text` #e8d5a0 copy): ratio ≈ 11.8:1 → AAA.
