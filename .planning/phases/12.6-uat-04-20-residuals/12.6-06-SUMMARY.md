**3. [Rule 1 — bug] TS7053 on PUERTA_POINT_BUY_SNAPSHOT index after `describe.each` body received `raceId: string`**

- **Found during:** Task 2 typecheck after adding the table-driven per-race suite.
- **Issue:** `compiledRaceCatalog.races.map(r => r.id)` yields `string[]` because `packages/data-extractor/src/contracts/race-catalog.ts:13` types the `.id` field as `z.string().regex(/^race:.../)`. The `describe.each` callback then received `raceId: string`, but `PUERTA_POINT_BUY_SNAPSHOT` is `Record<CanonicalId, PointBuyCurve>` (a branded template-literal union). `tsc` rejected the `[raceId]` index with TS7053 at two call sites.
- **Fix:** Cast `uniqueRaces` to `CanonicalId[]` at the dedupe boundary (`as CanonicalId[]`). This keeps the rules-engine contract strict while letting the test-side widened string from extractor validation flow through — the runtime values ARE valid `CanonicalId`s by regex, so the cast is sound (the Zod validation already ran at module load inside `compiledRaceCatalog` parse).
- **Files modified:** `tests/phase-12.6/ability-budget-per-race.spec.ts` (added `type { CanonicalId }` import + cast).
- **Commit:** `8f29937` (included in Task 2 commit).

**4. [Rule 2 — missing critical functionality] attributes-board-fail-closed.spec.tsx would have gone GREEN-on-wrong-path after snapshot populated**

- **Found during:** Task 2 pre-commit impact analysis — before running the suite.
- **Issue:** Plan 02's `attributes-board-fail-closed.spec.tsx` asserts the fail-closed callout renders when the user selects `race:human`. Under Plan 02's empty snapshot this worked because every race fell into the null branch. Under Plan 06's populated snapshot, `race:human` resolves to a valid curve → `isBlockedForMissingCurve` is false → callout is not rendered → those tests would fail without intervention. If I didn't proactively fix this, the spec suite would go red after commit; if I merely deleted the assertions, the "no snapshot entry ⇒ fail-closed" invariant would silently erode.
- **Fix:** Added symmetric inverse of the Plan 02 seed pattern — `savedHumanCurve = PUERTA_POINT_BUY_SNAPSHOT['race:human']; delete ...['race:human']` in `beforeEach`, and restoration in `afterEach`. This manufactures the no-entry condition locally per test while preserving the populated snapshot for all other test files. Vitest default per-file module isolation prevents leakage. Updated the file-level docstring to explain the pattern shift.
- **Files modified:** `tests/phase-12.6/attributes-board-fail-closed.spec.tsx` (added `PUERTA_POINT_BUY_SNAPSHOT` + `PointBuyCurve` imports; added save/delete in beforeEach; added restore in afterEach; updated docstring).
- **Commit:** `8f29937` (included in Task 2 commit).
