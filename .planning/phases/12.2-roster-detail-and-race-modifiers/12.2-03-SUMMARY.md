# Phase 12.2 Plan 03: Prestige Filter + AlignRestrict Decoder Summary

**One-liner:** Decode `CompiledClass.prerequisiteColumns.AlignRestrict` (NWN hex-bit mask) + introduce `BASE_CLASS_ALLOWLIST` fail-closed gate in `projectCompiledClass`, closing Phase 12.1 UAT Bug 3 at the adapter layer — Clérigo is no longer wrongly blocked on alignment, and five unreachable-at-L1 prestige-tagged classes are now correctly gated with a Spanish deferred-requirement label.

## Outcome

Closes Phase 12.1 UAT Bug 3. For an L1 Legal Bueno Humano foundation (`alignment:lawful-good`, no deity yet):

- **Clérigo** (`class:cleric`) — status `blocked` (not `illegal`). The remaining block comes from the overlay's `requiresDeity: true` plus `foundation.deityId === null`, which surfaces a "Requiere deidad" requirement row. Per D-04 "fail closed" this is acceptable for Phase 3 deity descope; the alignment gate no longer mis-blocks.
- **Alma Predilecta / Paladin Antiguos / Paladin Oscuro / Paladin Vengador / Artífice** — status `blocked` with the new Spanish deferred label `"Prerrequisitos específicos del servidor. Revisa las dotes, nivel de lanzador o atributos requeridos."` emitted by `projectCompiledClass` because their canonical ids fall outside `BASE_CLASS_ALLOWLIST`.
- **Guerrero** (`class:fighter`) — status `legal` (over-gating guard).
- **Shadowdancer** (`class:shadowdancer`) — status `blocked` with the preserved 12.1-01 deferred label `"Pendiente de dotes o habilidades de fases posteriores."` (prestige branch unchanged).

### Task 2 — GREEN decoder + projection (commit 852023b)
Extended `apps/planner/src/features/level-progression/class-fixture.ts`:

- Added `ALIGNMENT_BITS` readonly tuple table (9 entries, `0x001=LG..0x100=CE`).
- Exported `decodeAlignRestrict(mask, inverted)` — pure function. Returns `CanonicalId[]` (allowed set) when the mask has bits set; returns complement when `inverted === '1'`; returns `undefined` for null/empty/`0x00` input.
- Exported `decodePrerequisiteMaxLevel(value)` — pure function. Returns `number | null`; `'0'` → null.
- Exported `BASE_CLASS_ALLOWLIST: ReadonlyArray<CanonicalId>` with the 11 classic NWN base classes.
- Added private constants `DEFERRED_LABEL_PRESTIGE` and `DEFERRED_LABEL_UNVETTED_BASE`.
- Rewrote `projectCompiledClass` body:
  - `implementedRequirements` now decodes `AlignRestrict` + `InvertRestrict` via `decodeAlignRestrict`, then spreads `overlay.implementedRequirements ?? {}` LAST so overlay wins per-field.
  - `deferredRequirementLabels` is now a union: `!isBase` → `[DEFERRED_LABEL_PRESTIGE]` (preserves 12.1-01 behavior); `isBase && !BASE_CLASS_ALLOWLIST.includes(id)` → `[DEFERRED_LABEL_UNVETTED_BASE]` (fail-closed for unvetted base classes); allowlisted base → `[]`.
  - Every other field (description, exceptionOverrides, exclusiveClassIds, gainTable, hitDie, id, kind, label, minimumClassCommitment, tags) projects exactly as before.

## Long-term Follow-up

- **Extractor enrichment** — The `BASE_CLASS_ALLOWLIST` is a pragmatic escape, not a durable roster decision. The extractor should emit either (a) real `PreReqTable` decoding with BAB/class-level/feat prereqs, or (b) a `reachableAtLevelOne: boolean` field derived from the 2DA rows. When that lands, the allowlist shrinks to empty and the unvetted-base branch can be deleted. Tracked in `12.2-CONTEXT.md <deferred>`.
- **`decodePrerequisiteMaxLevel` is exported but currently unused by the projection adapter** — it was part of the plan's decoder-trio contract (`AlignRestrict` + `MaxLevel` + `PreReqTable`). For 12.2-03 the `MaxLevel` / `PreReqTable` signals are subsumed by the `BASE_CLASS_ALLOWLIST` fail-closed branch (class outside allowlist ⇒ deferred label ⇒ blocked, regardless of MaxLevel value). Leaving the helper exported anticipates the extractor enrichment above.
