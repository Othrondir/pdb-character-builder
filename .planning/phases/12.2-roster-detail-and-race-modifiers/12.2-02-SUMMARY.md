# Phase 12.2 Plan 02: Race Ability Modifier Pipeline Summary

**One-liner:** End-to-end race ability-modifier pipeline (CHAR-02): projection → store → pure rules-engine selector `applyRaceModifiers` → three-column Atributos board (Base | Racial | Final) → character sheet Final attributes, point-buy semantics preserved.

## Outcome

Closes Phase 12.1 UAT Bug 2 and the CHAR-02 structural gap inherited from Phase 3. `CompiledRace.abilityAdjustments` now flows from the extractor all the way through to the Atributos UI and the hoja-de-personaje stats panel. Picking Enano produces CON +2 / CHA −2; Elfo produces DEX +2 / CON −2; Humano shows all-zero Racial adjustments while still rendering the Racial column so the player knows it was evaluated.

### Task 2 — GREEN data + rules layer (commit a46bc9f)
- Added `packages/rules-engine/src/foundation/apply-race-modifiers.ts` exporting `applyRaceModifiers` + `BaseAttributes` / `RacialModifiers` / `FinalAttributes` types. Framework-agnostic: no React, no store reads, no browser APIs. Zero-fills when `racial === null`.

- Extended `CharacterFoundationStoreState` with `racialModifiers: Record<AttributeKey, number> | null`; added `lookupRacialModifiers` helper; `setRace` now sets both `raceId` + `racialModifiers`; `resetFoundation` and `setRace(null)` clear to null.

### Task 3 — GREEN UI layer (commit da29fff)
- Atributos board now renders three cells per attribute row: Base (+/- point-buy controls, unchanged) · Racial (signed read-only, `—` when no race) · Final (computed, prominent). Column header strip added (`aria-hidden="true"` — per-cell `aria-label` covers A11y). The existing (+mod) derivation is now computed from the Final score so the modifier matches real gameplay.
- CharacterSheet StatsPanel renders Final attributes via `applyRaceModifiers(foundationState.baseAttributes, foundationState.racialModifiers)`. AC, Fortaleza, Reflejos, Voluntad derived stats all switch from `baseAttributes` to Final.
- New scoped CSS: `.attributes-editor__cell`, `--base`, `--racial`, `--final`, `__column-headers*`. Uses `font-weight: 600` (not 700) to respect the phase-05.2 theme contract.
