<decisions>
## Implementation Decisions

### D-01 — Class picker layout (R1)

- **Decision:** Render a single scrollable picker column containing two `<section>` blocks (`Clases básicas` first, `Clases de prestigio` second), each with `<h3>` section heading and `aria-labelledby` wire-up. No tabs, no accordion collapse.
- **Rationale:** Matches SPEC acceptance (a) — two `<section>` containers. Preserves single-scroll idiom already used elsewhere in Construcción. Tabs hide half the roster behind a click; accordion adds state machinery for zero clarity win.
- **Expected touchpoints:** new `apps/planner/src/features/level-progression/class-picker.tsx` (extracted if currently inline in `level-sheet.tsx`) + CSS in `app.css` namespace `.class-picker__*`.

### D-02 — Class prestige prereq evidence (R1)

- **Decision:** Prereq copy renders inline beneath the disabled row in italic muted text (not tooltip, not badge). Copy shows **threshold only**, not live progress: `Requiere BAB ≥ 5` / `Requiere 8 rangos de Sigilo` / `Requiere dote: Esquiva`. At L1 all prestige rows disabled with `Disponible a partir del nivel 2`.
- **Rationale:** Tooltips hide info on mobile + keyboard; inline copy is accessible and self-describing. "Progress toward unlock" (e.g. `BAB 3/5`) requires live BAB selector + polish work beyond 12.4 correctness scope — deferred.
- **Expected touchpoints:** new `reachableAtLevelN(build, classId, level)` pure helper in `packages/rules-engine/src/progression/prestige-gate.ts` returning `{ reachable: boolean, blockers: Array<{kind: 'bab'|'skill-rank'|'feat'|'class-level'|'l1', threshold: number|string, label: string}> }`. UI reads `blockers[0].label` for copy.
- **Extractor dependency:** SPEC notes extractor does not yet surface BAB / skill-rank / feat prereqs for prestige gating. `BASE_CLASS_ALLOWLIST` (from Phase 12.2-03) remains the fail-closed escape hatch. Plan MUST audit which prestige classes have decoded prereqs today and stub the remainder with `reachable: false, blockers: [{kind: 'l1', label: 'Requisitos en revisión'}]` until extractor enrichment lands (tracked as deferred).

### D-03 — Dotes selectability visual treatment (R5)

- **Decision:** Four states resolve inside the feat-board selector and drive row CSS class + inline copy:
  - `selectable` → default row (`.feat-picker__row`), cursor:pointer, full opacity, `aria-disabled=false`.
  - `blocked-prereq` → `.feat-picker__row--blocked`, opacity 0.55, cursor:not-allowed, inline italic copy below label (`Requiere dote: Esquiva`), pill badge right-aligned (`Bloqueada`).
  - `blocked-already-taken` → same blocked class + pill badge (`Tomada en N{level}`).
  - `blocked-budget` → same blocked class + pill badge (`Sin slots`).
- **Visibility lock:** SPEC R5 forbids `display: none` on blocked rows. CSS contract enforced via test (`expect(blockedRow).toBeVisible()` in RTL).
- **Rationale:** Opacity + cursor + inline reason + badge is redundant-but-accessible. Covers keyboard users (badge text), screen readers (inline copy), and glance users (opacity). Tooltips rejected — hidden from non-pointer users.
- **Expected touchpoints:** `apps/planner/src/features/feats/selectors.ts` (add `rowState` + `blockedReason` to `FeatOptionView`), `feat-board.tsx` (render), `app.css` `.feat-picker__*` namespace.

### D-04 — Dotes collapse-on-complete interaction (R5)

- **Decision:** When the per-level `chosen === slots` predicate flips true, the feat-board body swaps to a summary card listing chosen feats (one per line) + a right-aligned `Modificar selección` TextButton (underlined link-style). Click on TextButton restores the full list with chosen feats highlighted via `.feat-picker__row--chosen` (subtle gold outline). No animated collapse — instant swap matches NWN1 static UI feel.
- **Rationale:** Summary card reduces visual load at completion without hiding state. Re-expand via named action keeps discoverability (vs expanding on hover). Highlighting chosen rows in the re-expanded list preserves orientation.
- **Expected touchpoints:** `feat-board.tsx` conditional render, new `<FeatSummaryCard>` component, CSS.

### D-05 — Feat-family secondary picker (R7)

- **Decision:** Inline expander under the family row (not modal, not nested side column). Expander is a `<fieldset>` with `<legend>` copying the family label, containing a radio-button list of valid targets. Already-taken targets get the same `blocked-already-taken` treatment from D-03. Esc key collapses the expander; Enter confirms the focused radio. Max-height on the expander (`max-height: 320px; overflow-y: auto;`) mirrors `.feat-picker__list` scroll idiom.
- **Rationale:** Modal focus-trap is overkill for a 2-3-click scoped pick and breaks context (user loses sight of the full feat list). Inline expander keeps the family + its chosen target visible in one column. Nested side column conflicts with R9 scroll relocation (list column already owns scroll).
- **Expected touchpoints:** new `apps/planner/src/features/feats/feat-family-expander.tsx`, extended `FeatOptionView` with `family: FeatFamilyContract | null` field, extractor `parameterizedFeatFamily` emission in `packages/data-extractor/src/assemblers/feat-assembler.ts`.
- **Persisted id format:** target-qualified (e.g. `feat:skill-focus:animal-empathy`) — fits existing `BuildDocument` Zod schema per SPEC constraint.

### D-06 — `LevelEditorActionBar` shape (R2)

- **Decision:** New component `apps/planner/src/features/level-progression/level-editor-action-bar.tsx` rendered inside `.level-sheet` as a sticky footer (`position: sticky; bottom: 0;` with `border-top: 1px solid var(--color-panel-edge);`). Renders a single `NwnButton` whose label + disabled state derive from the completion predicate:
  - Enabled `Continuar al nivel {N+1}` → all slots filled.
  - Disabled `Faltan {N} dotes que asignar en este nivel` → feat slots unfilled.
  - Disabled `Faltan {N} puntos de habilidad por gastar` → skill-point budget unspent.
  - Plural-aware Spanish copy (`1 dote` / `2 dotes`).
- **Coexistence:** Rail click remains a navigation escape hatch (user can still jump to any level). Advance button is the **primary completion affordance**; rail is the secondary freeform navigator. No deprecation of rail click.
- **Hidden on L16:** terminal level has no `N+1` — component returns `null` when `level === 16`.
- **Rationale:** Sticky footer matches the `ActionBar` idiom already used in Atributos (`Aceptar` button pattern referenced in UAT F2). Self-describing disabled label removes tooltip need.
- **Expected touchpoints:** new component + CSS `.level-editor__action-bar`, reads from `selectLevelCompletionState(build, level)` in `level-progression/selectors.ts`.

### D-07 — `per-level-budget` selector module layout (R3)

- **Decision:** Single module `packages/rules-engine/src/progression/per-level-budget.ts` exporting:
  ```ts
  export function computePerLevelBudget(
    build: BuildState,
    level: number,
    classCatalog: ClassCatalog,
    raceCatalog: RaceCatalog,
  ): PerLevelBudget
  ```
  Returns `{ featSlots: { general, classBonus, raceBonus, total, chosen, remaining }, skillPoints: { budget, spent, remaining } }`. Pure function — no `window`, no `fetch`, no store import.
- **Canon sources (per SPEC constraint chain):** manual override > Puerta snapshot > NWN1 EE canon:
  - Humano L1 bonus feat, Guerrero L1/L2/+2 cadence, Mago L1+L5+L10+L15, Monje L1/L2/L6, Pícaro L10/L13/L16, Guardián enano bonus feat, prestige-class bonus feats.
  - L1 skill-point quadruple (base × 4 + INT × 4), Pícaro/Explorador INT-based per-level bonuses, prestige-class transition SP.
- **Memoization:** selector-level via zustand `useShallow` + structural equality on `build.classByLevel[level]` + `build.abilityScores.int`. No internal memo map inside the helper — callers memoize.
- **Perf gate:** vitest bench run asserts p95 < 1ms on L16 full-build. If gate misses, add internal memoization behind a `lru-cache` wrapper before merge.
- **Rationale:** Single file keeps the feat-slot + skill-point math co-located with the level context they share; splitting to two files forces callers to import both every time.
- **Expected touchpoints:** new `per-level-budget.ts` + barrel export in `packages/rules-engine/src/progression/index.ts` + fixture spec `tests/phase-12.4/per-level-budget.fixture.spec.ts` + bench spec.

### D-08 — L1 neutral state + auto-seed removal (R6 + X1)

- **Decision:** Remove auto-seed in `level-progression/store.ts` — initial `classByLevel` starts as `{}` (or `classByLevel[1] = null` if the shape demands a placeholder). Sub-step ✓ predicates land in `level-progression/selectors.ts`:
  - `isClaseLevelComplete(build, level) = build.classByLevel[level] != null`
  - `isHabilidadesLevelComplete(build, level) = computePerLevelBudget(...).skillPoints.remaining === 0`
  - `isDotesLevelComplete(build, level) = computePerLevelBudget(...).featSlots.chosen === computePerLevelBudget(...).featSlots.total`
- **Rail tile label:** L1 rail tile renders blank label (no `Explo…` ellipsis) until class is selected. Existing `.level-rail__abbrev` CSS already handles overflow ellipsis — the blank-label case just means the abbrev source string is empty.
- **Rationale:** SPEC R6 locks the target — this decision names the exact predicate owners.
- **Expected touchpoints:** `level-progression/store.ts` (drop auto-seed), `level-progression/selectors.ts` (new predicates), `level-progression/build-progression-board.tsx` (rail tile label binding).

### D-09 — Habilidades class/transclase split + cost copy (R4)

- **Decision:** `skill-board.tsx` renders two `<section>` blocks (`Habilidades de clase · coste 1 pt/rango`, `Habilidades transclase · coste 2 pts/rango`) with `<h3>` headings ≥ 12px. Skills bucketed by the active class's `classSkills` Set from compiled-classes. `Solo entrenada` stays as a per-row badge orthogonal to section membership.
- **Cost copy legibility:** explicit 12px floor encoded in CSS (`.skill-board__section-heading { font-size: 12px; }` or a shared `--font-size-label` token ≥ 12px).
- **Rationale:** Matches SPEC R4 acceptance verbatim. Muted tag rejected (SPEC evidence: "almost invisible").
- **Expected touchpoints:** `apps/planner/src/features/skills/skill-board.tsx` + `selectors.ts` (add `sectionId: 'class' | 'cross-class'` to row view) + CSS `.skill-board__*`.
