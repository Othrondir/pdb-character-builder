# Phase 12.4 — UI Design Contract

> Visual and interaction contract for the Construcción correctness + clarity phase (9 requirements, R1..R9). Generated by gsd-ui-researcher from SPEC.md + CONTEXT.md (D-01..D-10). Scope is correctness + clarity, NOT polish — no animation changes, no icon system changes, no theme-token additions. Downstream: gsd-planner consumes tokens + copy strings, gsd-executor consumes visual source of truth.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (hand-rolled CSS + CSS variables) |
| Preset | not applicable |

## Spacing Scale

8-point scale (locked in `tokens.css`):

| Token | Value | CSS var | Usage (12.4-specific) |
|-------|-------|---------|-----------------------|
| xs | 4px | `--space-xs` | Inline icon/badge gaps, rail sub-step internal gap, pill-badge padding, feat row vertical padding |
| sm | 8px | `--space-sm` | Feat row horizontal gap, skill section header bottom margin, expander radio gap |
| md | 16px | `--space-md` | Panel default padding, ActionBar padding, section vertical gap inside picker column, summary card body padding |
| lg | 24px | `--space-lg` | Section outer padding, `<FeatSummaryCard>` padding, detail panel padding |
| xl | 32px | `--space-xl` | Major picker-to-detail column gap, progression board grid gap |
| 2xl | 48px | `--space-2xl` | Not used in 12.4 |
| 3xl | 64px | `--space-3xl` | Not used in 12.4 |

**Exceptions (enumerated, not "other"):**
- Touch-target minimum 44px on `NwnButton` (existing token: `min-height: 44px`). `LevelEditorActionBar` button inherits this.
- Feat row icon/badge cluster gap: 4px (xs).
- Pill badge internal padding: 2px vertical × 8px horizontal — explicitly non-token because badges are sub-row ornaments, not layout primitives. Encoded as literal in `.feat-picker__pill` rule, commented inline.
- Feat-family expander `max-height: 320px` (D-05) — literal, non-token (mirrors existing `.feat-picker__list` scroll idiom).
- Section heading minimum font-size floor 12px (R4 explicit acceptance) — NOT a space token, encoded in typography block.

## Typography

Four sizes × two weights. All values already present in `tokens.css` / `app.css`; **12.4 adds no new font sizes or weights**.

| Role | Size | Weight | Line Height | Font family | Usage in 12.4 |
|------|------|--------|-------------|-------------|---------------|
| Display (uppercase heading) | 18px | 600 | 1.2 | `--font-display` | `SelectionScreen` title bars (`CLASE`, `HABILIDADES`, `DOTES`), `Resumen` heading |
| Subheading (section) | 16px | 600 | 1.2 | `--font-display` | `<section>` headings `Clases básicas`, `Clases de prestigio`, `Habilidades de clase · coste 1 pt/rango`, `Habilidades transclase · coste 2 pts/rango`, `Dotes del nivel N` panel header, `<FeatSummaryCard>` heading |
| Body | 14px | 400 (regular) or 600 (emphatic) | 1.5 | `--font-body` | Feat row labels, class row labels, level rail labels, prereq inline copy (italic), summary card chosen-feat list, `LevelEditorActionBar` button label |
| Label (small / metadata) | 12px | 400 | 1.4 | `--font-body` | Section heading cost hint (`· coste 1 pt/rango`) — **MUST render at ≥12px per R4 acceptance**, pill badge copy (`Bloqueada`, `Tomada en N{level}`, `Sin slots`, `Disponible a partir del nivel 2`), feat description meta, rail tile abbrev, skill-board row meta-inline |

**Weights allowed:** 400 (regular) and 600 (semibold). No 500, no 700 — the single `font-weight: 700` in `app.css:113` is an existing token violation flagged in STATE.md session notes; **12.4 MUST NOT introduce new 700 weights**. If `<LevelEditorActionBar>` inherits `nwn-button` weight 600, no change needed.

**Italic usage — reserved:** only for inline prereq evidence copy beneath disabled prestige rows (D-02) and inline blocked-reason copy beneath disabled feat rows (D-03). Italic signals "diagnostic / evidence" text, not decoration. Never italic for primary labels.

**Uppercase + letter-spacing:** reuse existing pattern — `text-transform: uppercase; letter-spacing: 0.10em;` on all `<h3>` section headings. Spanish accented characters render correctly at that letter-spacing (verified in existing `.skill-sheet__group h3`).

**No custom numerals token.** Attributes board already uses `font-variant-numeric: lining-nums tabular-nums` for counters — `Dotes del nivel N: {chosen}/{slots}` and `Puntos de habilidad: {spent}/{budget}` counters MUST apply the same variant so digits are tabular.

## Color

60/30/10 palette locked in `tokens.css`. **12.4 adds no color tokens.**

| Role | Value | CSS var | Usage |
|------|-------|---------|-------|
| Dominant (60%) — surface | `#0a0a0a` | `--color-surface` | Page background, picker column background, scrollbar track |
| Secondary (30%) — panel | `#141010` | `--color-panel` | `planner-panel`, `level-sheet`, `<FeatSummaryCard>` background, detail panel surface, sticky footer background |
| Accent (10%) — gold | `#c8a84e` | `--color-gold` | **Reserved-for list below** |
| Accent-variant (muted) | `#7a6528` | `--color-gold-dark` | Scrollbar thumb default state only |
| Text | `#e8d5a0` | `--color-text` | Default body copy, active-row text, button labels |
| Text muted | `#8b7355` | `--color-text-muted` | Metadata (skill category tag pre-split, pill-badge text on blocked rows, meta-inline in skill rows, rail tile pending color) |
| Panel edge | `rgba(200, 168, 78, 0.35)` | `--color-panel-edge` | Inner frame, sticky footer `border-top`, section heading underline, option-list right border |
| Selection | `rgba(200, 168, 78, 0.18)` | `--color-selection` | `is-selected`, `is-active`, `is-expanded` row backgrounds (feat-picker chosen row, class-picker active row) |
| Selection border | `rgba(200, 168, 78, 0.75)` | `--color-selection-border` | Border for selected / chosen-highlighted rows — also gives `<FeatSummaryCard>` chosen-feat outline, `.feat-picker__row--chosen` gold outline on re-expand |
| Blocked border | `rgba(200, 168, 78, 0.55)` | `--color-blocked` | `is-blocked` border on feat rows (`blocked-prereq`, `blocked-already-taken`, `blocked-budget`), prestige-blocked class rows |
| Destructive | `#7a1f1f` | `--color-destructive` | Reserved — 12.4 introduces no new destructive surfaces. Existing `is-illegal` rows (skill / level / feat) keep destructive border already in `app.css`. |

**Accent gold (`--color-gold`) reserved strictly for:**
1. Primary frame borders (`NwnFrame`, `.planner-panel`, title-bar bottom rule).
2. `NwnButton--primary` border + focus-visible outline.
3. Selection-state border + title underline (via `--color-selection-border` derivative).
4. `SelectionScreen` title-bar bottom rule (`border-bottom: 1px solid var(--color-gold)`).
5. **New in 12.4:** pill-badge border on `blocked-*` feat rows (1px solid `--color-blocked`, which derives from gold). Pill-badge text uses `--color-text-muted`.
6. **New in 12.4:** `.feat-picker__row--chosen` highlight outline on re-expanded feat list (1px solid `--color-selection-border`).
7. **New in 12.4:** `LevelEditorActionBar` sticky-footer `border-top: 1px solid var(--color-panel-edge)` (NOT gold — muted edge matches existing `.creation-stepper__bottom` idiom).

**Explicitly NOT using accent:**
- Body copy (stays `--color-text`).
- Unselected / selectable feat rows (stay panel background + muted border, not gold).
- Hover state alone is not enough to earn gold — only selection / chosen / focused states get the gold treatment.

**Contrast verification (WCAG AA at 14px):**
- `--color-text` (#e8d5a0) on `--color-panel` (#141010) → contrast ratio 10.9:1 ✓ (existing, unchanged).
- `--color-text-muted` (#8b7355) on `--color-panel` (#141010) → contrast ratio 4.6:1 ✓ (existing).
- Blocked row at opacity 0.55 against panel background: effective text color becomes ≈#77664b on #141010 → contrast ≈3.7:1 at 14px → **FAILS AA body**. **Mitigation:** inline italic prereq copy and pill-badge copy MUST render at full opacity (no opacity inheritance). Only row label + description inherit the 0.55 opacity. Plan + executor MUST encode `opacity: 1` on the `.feat-picker__row--blocked .feat-picker__reason` and `.feat-picker__row--blocked .feat-picker__pill` selectors to defeat cascade.

## Layout & Structure

### R1 — Class picker (D-01)
```
<section class="class-picker">
  <section aria-labelledby="class-picker__basic">
    <h3 id="class-picker__basic">Clases básicas</h3>
    <ul class="class-picker__list">
      <li><button class="class-picker__row">...</button></li>
      ... (11 base class rows, one per BASE_CLASS_ALLOWLIST entry)
    </ul>
  </section>
  <section aria-labelledby="class-picker__prestige">
    <h3 id="class-picker__prestige">Clases de prestigio</h3>
    <ul class="class-picker__list">
      <li><button class="class-picker__row is-blocked" aria-disabled="true">
        <span class="class-picker__label">Asesino Oscuro</span>
        <em class="class-picker__reason">Requiere BAB ≥ 5</em>
      </button></li>
      ... (one row per prestige class)
    </ul>
  </section>
</section>
```
- Single scrollable column (D-01). No tabs, no accordion.
- `aria-labelledby` wires each `<section>` to its `<h3>`.
- Two sections separated by `var(--space-lg)` vertical gap.
- Prestige blocked rows: `opacity: 0.55` on row, `cursor: not-allowed`, italic reason copy at full opacity (D-02).
- At L1, every prestige row is blocked; reason copy is `Disponible a partir del nivel 2`.

### R2 — Level editor footer (D-06)
```
<footer class="level-editor__action-bar">
  <!-- sticky: position: sticky; bottom: 0; border-top: 1px solid var(--color-panel-edge); -->
  <button class="nwn-button nwn-button--primary">
    Continuar al nivel 2
  </button>
</footer>
```
- Single-button footer. Pinned to `level-sheet` scrolling container bottom.
- Returns `null` (renders nothing) when `level === 16`.
- Three dynamic label states (see Copywriting Contract).
- Disabled state: `opacity: 0.45`, `cursor: not-allowed` (inherits `.nwn-button:disabled`).
- No secondary/cancel button. Rail click remains the navigation escape hatch.

### R4 — Skill board split (D-09)
```
<section class="skill-board">
  <section aria-labelledby="skill-board__class">
    <h3 id="skill-board__class" class="skill-board__section-heading">
      Habilidades de clase <span class="skill-board__cost-hint">· coste 1 pt/rango</span>
    </h3>
    <ul class="skill-board__rows">...</ul>
  </section>
  <section aria-labelledby="skill-board__cross-class">
    <h3 id="skill-board__cross-class" class="skill-board__section-heading">
      Habilidades transclase <span class="skill-board__cost-hint">· coste 2 pts/rango</span>
    </h3>
    <ul class="skill-board__rows">...</ul>
  </section>
</section>
```
- Heading + cost hint on the same line; hint is `<span>` with muted color but **font-size MUST remain ≥12px** (R4 acceptance line).
- `Solo entrenada` badge stays per-row, orthogonal to section membership.
- Rank cap copy (`Tope: 4` / `Tope: 2`) stays per-row on the right edge (existing pattern).

### R5 — Dotes picker + summary card (D-03, D-04)
Expanded state:
```
<section class="feat-picker">
  <header class="feat-picker__header">
    <h3>Dotes del nivel 2: 1/2</h3>
  </header>
  <ul class="feat-picker__list">  <!-- owns overflow-y: auto (R9) -->
    <li><button class="feat-picker__row">
      <span class="feat-picker__label">Esquiva</span>
    </button></li>
    <li><button class="feat-picker__row feat-picker__row--blocked" aria-disabled="true">
      <span class="feat-picker__label">Soltura con un arma</span>
      <em class="feat-picker__reason">Requiere BAB ≥ 1</em>
      <span class="feat-picker__pill">Bloqueada</span>
    </button></li>
    ... (blocked rows stay in DOM — never display:none per R5)
  </ul>
</section>
```
Collapsed state (chosen === slots):
```
<section class="feat-picker feat-picker--complete">
  <header class="feat-picker__header">
    <h3>Dotes del nivel 2: 2/2</h3>
  </header>
  <article class="feat-summary-card">
    <ul class="feat-summary-card__list">
      <li>Esquiva</li>
      <li>Soltura con un arma (Espada larga)</li>
    </ul>
    <button class="feat-summary-card__modify nwn-button--auxiliary">
      Modificar selección
    </button>
  </article>
</section>
```
- `<FeatSummaryCard>` sits inside the same `<section class="feat-picker">` so the header counter stays visible.
- Card background: `var(--color-panel)`; card border uses existing `.nwn-frame` visual idiom (2px gold + inner edge); padding `var(--space-lg)`.
- `Modificar selección` rendered as `nwn-button--auxiliary` (underlined-link visual is not our idiom; auxiliary = muted + 400 weight matches NWN1 static feel, SPEC constraint).
- Re-expand: click on `Modificar selección` restores the list. Chosen feats inside the re-expanded list render with `.feat-picker__row--chosen` → 1px gold-selection outline on the row.

```
<li>
  <button class="feat-picker__row feat-picker__row--family" aria-expanded="false">

    <div class="feat-family-expander__options">  <!-- max-height: 320px; overflow-y: auto; -->
      <label><input type="radio" name="family"> Trato con los animales</label>

      <label><input type="radio" name="family" disabled> Buscar <span class="feat-picker__pill">Ya seleccionada</span></label>
    </div>
  </fieldset>
</li>
```
- Inline expander, NOT modal (D-05).
- Esc collapses the expander; Enter confirms the focused radio.
- Max-height 320px with internal scroll; mirrors `.feat-picker__list` overflow idiom.
- Already-taken targets get the same `blocked-already-taken` badge treatment from D-03.

### R9 — Dotes scroll relocation
- `.feat-picker__list { overflow-y: auto; }` — scroll lives on the list column, NOT the two-column wrapper.
- Description panel (`.detail-panel`) already owns its own internal scroll (existing code). Verify class-picker + race-picker CSS follow the same list-owns-scroll pattern (audit task inside plan 12.4-02).
