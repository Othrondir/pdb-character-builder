# Phase 12.4-07: R5 Dotes Four Selectability States + Slot Counter + Collapse-on-Complete Summary

## One-liner

Dotes picker now resolves four row states (`selectable` / `blocked-prereq` / `blocked-already-taken` / `blocked-budget`) with visibility lock (blocked rows stay in DOM), header counter `Dotes del nivel N: {chosen}/{slots}` fed by `computePerLevelBudget`, and collapse-on-complete swap to `<FeatSummaryCard>` with a `Modificar selecciĂłn` re-expand button that highlights chosen rows via `.feat-picker__row--chosen` gold outline.

## Problem

SPEC R5 / FEAT-01: the ConstrucciĂłn Dotes route rendered a flat filtered list (via `getEligibleFeats`) with no per-row selectability states, no header slot counter, no collapse-on-complete, and no visibility for blocked-prereq feats. Users couldn't see "Movilidad is blocked because you haven't taken Esquiva yet" â€” the row simply didn't appear â€” breaking forward-planning. There was no evidence the user had filled the level's feat budget; they had to guess by clicking the rail. The CONTEXT D-03/D-04 decisions lock the four-state matrix + visibility lock + collapse interaction as the correctness + clarity fix.

## Approach

Three layers:

### 1. Pure selector extension

`apps/planner/src/features/feats/selectors.ts` gained three helpers and a refactored `selectFeatBoardView`:

- **`formatBlockedReason(check: PrerequisiteCheck) â†’ string`** â€” routes each `PrerequisiteCheck.type` (ability / bab / feat / or-feats / skill singular+plural / class-level singular+plural / fort-save / character-level) to the matching Spanish template under `shellCopyEs.feats.blockedReasons`. Ready-to-render strings; no UI-layer interpolation.
- **`findAlreadyTakenAtLevel(featState, featId, activeLevel) â†’ number | null`** â€” scans `featState.levels` for the earliest level at which the feat was picked (class-bonus or general slot), excluding `activeLevel` itself so the user's current pick stays selectable (toggle-off allowed).
- **`buildSnapshotForBudget(...) â†’ BuildSnapshot`** â€” storeâ†’BuildSnapshot boundary adapter that pipes zustand state into the pure `computePerLevelBudget` (12.4-03). Keeps `packages/rules-engine` framework-agnostic per CLAUDE.md Prescriptive Shape.
- **`resolveFeatRowState({ feat, buildState, featState, activeLevel, isChosenAtLevel, budget }) â†’ { rowState, blockedReason }`** â€” priority-ordered: already-taken > prereq > budget > selectable.

`selectFeatBoardView` now walks `compiledFeatCatalog.feats` directly (with `preReqEpic` + sentinel filters from 12.4-01) instead of consuming `getEligibleFeats`. Every feat in the class-bonus pool (list=1/2) OR the general pool (`allClassesCanUse=true` OR list=0 for the active class) produces a `FeatOptionView` â€” including blocked-prereq rows â€” preserving the SPEC R5 visibility-lock contract.

### 2. UI components

- **New `<FeatSummaryCard>`** (`apps/planner/src/features/feats/feat-summary-card.tsx`): `NwnFrame` wrapping a `<ul>` of chosen-feat labels + an `NwnButton variant="auxiliary"` labeled `Modificar selecciĂłn`. Instant swap â€” no animation â€” matches NWN1 static UI feel (CONTEXT D-04 rationale).
- **`<FeatBoard>` update**: panel header renders `boardView.counterLabel` (`Dotes del nivel N: {chosen}/{slots}`). Local `isEditingCompleted` state (reset on `activeLevel` change via `useEffect`) toggles between `<FeatSheet>` (full list) and `<FeatSummaryCard>` (collapsed). Predicate: `counters.slots > 0 && counters.chosen >= counters.slots && !isEditingCompleted`.
- **`<FeatSheet>` refactor**: `OptionList` replaced with a bespoke `<FeatPickerRow>` button carrying `data-feat-id`, `aria-disabled={blocked ? 'true' : 'false'}`, classes `.feat-picker__row [--blocked] [--chosen] [--focused]`, inline italic `<em class="feat-picker__reason">` when `blockedReason.reasonLabel` is non-null, and right-aligned `<span class="feat-picker__pill">` when `blockedReason.pillLabel` is non-null. The `<ul class="feat-picker__list">` wrapper preserves the 12.4-02 scroll-on-list contract.

### 3. Copy + CSS

- `shellCopyEs.feats` gained `slotCounterTemplate`, `blockedPills` (prereq / alreadyTakenTemplate / budget), `blockedReasons` (12 templates covering every `PrerequisiteCheck.type` branch), and `modifySelectionLabel`. All Spanish-first per SPEC constraint.
- `app.css` append-only block (after 12.4-06's `.class-picker__*`): `.feat-picker__header`, `.feat-picker__counter` (tabular-nums), `.feat-picker__row` + `--blocked` (opacity 0.55) + `--chosen` (gold selection border) + `--focused`, `.feat-picker__label`, `.feat-picker__reason`, `.feat-picker__pill`, the cascade-defeat rule `.feat-picker__row--blocked .feat-picker__reason, .feat-picker__row--blocked .feat-picker__pill { opacity: 1 }`, focus-visible, `.feat-summary-card` + `__list` + `__item` + `__modify`. Zero hardcoded hex colors â€” all via `var(--color-*)` tokens.
