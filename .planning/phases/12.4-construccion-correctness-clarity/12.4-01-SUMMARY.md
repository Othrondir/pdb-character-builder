# Phase 12.4 Plan 01: R8 Extractor + UI Sentinel Filter Summary

Closes SPEC R8 (D-10 Wave 1): eliminates sentinel rows (`DELETED`, `***DELETED***`, `UNUSED`, `PADDING`, `DELETED_*`, `****`) from the compiled catalogs via a shared `isSentinelLabel` predicate wired into all four extractor assemblers (feat/skill/class/race) and the UI feat selector as belt-and-braces. Regenerates the shipped `compiled-feats.ts` artifact so the Dotes picker stops rendering the two confirmed `DELETED` rows at sourceRow 385 + 403.

## What Shipped

### 1. Shared sentinel predicate (producer contract)

New module `packages/data-extractor/src/lib/sentinel-regex.ts` exports:

```typescript
export const SENTINEL_REGEX =
  /^(\s*\*{3,}\s*(DELETED|UNUSED|PADDING)\s*\*{3,}\s*|\s*(DELETED|UNUSED|PADDING)(_.*)?\s*|\s*\*{3,}\s*)$/i;

export function isSentinelLabel(label: string | null | undefined): boolean {
  if (label === null || label === undefined) return false;
  const trimmed = label.trim();
  if (trimmed.length === 0) return false;
  return SENTINEL_REGEX.test(trimmed);
}
```

The regex covers three sentinel families in one pattern:
- `***DELETED***` / `***UNUSED***` / `***PADDING***` (with optional surrounding whitespace).
- Bare `DELETED` / `UNUSED` / `PADDING`, plus `DELETED_EPIC` / `PADDING_RESERVED` underscore-suffix families.
- Standalone `***`, `****`, `*****`, etc. (3+ stars alone).

All matches are case-insensitive. Null, undefined, and empty strings return false.

### 2. Four assembler guards (fail-closed at producer)

Each of the four assemblers gains an `isSentinelLabel(label)` check immediately after the existing missing-label early-continue:

| Assembler | Guard count | Insertion site |
|-----------|-------------|----------------|
| `feat-assembler.ts` | 2 | After `!label || label === '****'` in id-mapping loop; after `displayLabel` resolution in the emission loop |
| `skill-assembler.ts` | 1 | After `!label` check, before `HideFromLevelUp` filter |
| `class-assembler.ts` | 1 | After `PlayerClass !== '1'` + missing-label checks, before `canonicalId` |
| `race-assembler.ts` | 1 | After `PlayerRace !== '1'` + missing-label checks, before `canonicalId` |

Matched rows log a `warnings.push(`{Catalog} row ${rowIndex}: sentinel label '${label}' — skipped`)` audit trail and `continue` — the row never reaches `canonicalId` / emission.

### 3. UI belt-and-braces filter (consumer contract)

`apps/planner/src/features/feats/selectors.ts::selectFeatBoardView` now filters both feat arrays before view mapping:

```typescript
eligibleClassFeats: eligible.classBonusFeats
  .filter((f) => !isSentinelLabel(f.label))
  .map((f) => mapToOptionView(f, selectedClassFeatId)),
eligibleGeneralFeats: eligible.generalFeats
  .filter((f) => !isSentinelLabel(f.label))
  .map((f) => mapToOptionView(f, selectedGeneralFeatId)),
```

The filter defends against future catalog contamination (hand-edited `compiled-feats.ts`, partial extractor runs) and means the Dotes picker can never render a sentinel row even if the artifact regresses.

Both objects were excised (22 lines removed). Cross-reference grep confirmed zero catalog references to either id (no `PREREQFEAT1`, `orReqFeats`, `classFeatLists` entries), so removal is schema-safe. Zod validation of the regenerated catalog passes.

A fresh `pnpm extract` run from a normal workspace (with `nwsync` files present) will reproduce the same scrubbed artifact automatically via the four new assembler guards — this manual edit is a one-shot fix for the already-shipped artifact in the isolated parallel worktree.
