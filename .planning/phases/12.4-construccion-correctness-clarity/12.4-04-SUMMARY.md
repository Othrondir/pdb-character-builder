# Phase 12.4 Plan 04: R6 + X1 L1 Neutral Sub-Steps Summary

**One-liner:** Close SPEC R6 + X1 regression by removing the `level-sub-steps.tsx:31` auto-`'complete'` fallback that pre-greened every L1 sub-step chip before user interaction; introduce three pure boolean predicates (`isClaseLevelComplete` / `isHabilidadesLevelComplete` / `isDotesLevelComplete`) in `level-progression/selectors.ts` that compose Wave 1 plan 12.4-03's `computePerLevelBudget` via planner-side catalog adapters + a new `BuildSnapshot` composer; extend `StepperStepStatus` with a fifth `'ready'` variant (enabled + neutral) so chips can be clickable-but-neutral without inheriting origin-stepper's disabled-when-pending semantics; wire `data-substep="class|skills|feats"` onto each sub-step button so RTL specs can assert completion state deterministically; extend `vitest.config.ts::environmentMatchGlobs` with the Wave 0 `tests/phase-12.4/**/*.spec.tsx → jsdom` mapping.

## Outcome

Closes SPEC R6 (L1 sub-steps render neutral until user affirms) and X1 regression (pre-green ✓ on Clase/Habilidades/Dotes):

- **Store contract preserved:** `createInitialProgressionState` already returns `levels` with `classId: null` (progression-fixture.ts L37-38). No auto-seed fix needed; RTL spec Suite A (A1 + A2) is a permanent forward-looking lock.
- **Three pure predicates exported** from `apps/planner/src/features/level-progression/selectors.ts`:
  - `isClaseLevelComplete(progression, level)` — true iff `classId !== null` at this level.
  - `isHabilidadesLevelComplete(progression, foundation, feats, skills, level)` — true iff `skillPoints.budget > 0 && skillPoints.remaining === 0`. Empty budget ⇒ false ("earned not defaulted").
  - `isDotesLevelComplete(progression, foundation, feats, skills, level)` — true iff `featSlots.total > 0 && featSlots.chosen === featSlots.total`. Empty slot count ⇒ false.
- **Sub-step UI wiring:** `components/shell/level-sub-steps.tsx` now subscribes to the four planner stores, calls the predicates per chip, and resolves each chip's status to `'active' | 'complete' | 'ready'`. `data-substep="class|skills|feats"` lands on every chip button (piped through `components/shell/stepper-step.tsx`'s new optional `dataSubStep` prop).
- **StepperStepStatus union extended** with `'ready'` (enabled + neutral + no icon). Origin-stepper's `'pending'` semantics (disabled + greyed) preserved — pending stays pending for Alineamiento-before-Raza / Atributos-before-Alignment gates.
- **Wave 0 infra applied:** `vitest.config.ts::environmentMatchGlobs` extended with `['tests/phase-12.4/**/*.spec.tsx', 'jsdom']`. `*.spec.ts` under phase-12.4 stays node (sentinel / per-level-budget / bench specs are pure helpers).
- **Full Vitest suite: 563/564 passing** (+12 new cases; zero regressions; the sole remaining failure is the pre-existing `tests/phase-05.2/theme-contract.spec.ts > does NOT contain font-weight: 700` which persists from pre-plan baseline 551/552 and is documented in 12.4-03's deferred issues).
- **Typecheck clean:** `corepack pnpm typecheck` exits 0.
