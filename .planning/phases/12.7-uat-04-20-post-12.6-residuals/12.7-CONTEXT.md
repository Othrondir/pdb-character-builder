## Implementation Decisions

### Mount Site (R1 — F7 BLOCKER)
- **D-01:** `LevelEditorActionBar` hoists to `apps/planner/src/components/shell/creation-stepper.tsx` as a sibling of the `<CenterContent>` host — rendered at the bottom of the stepper frame so it persists across Clase / Habilidades / Dotes sub-step renders. Mount is conditional on `expandedLevel !== null` (only when a Progresión level is active); returns `null` otherwise.
- **D-02:** Stable React key `key={expandedLevel}` (NOT `key={activeLevelSubStep}`) so the action bar does NOT remount when the user transitions Clase → Habilidades → Dotes within the same level. Acceptance criterion in SPEC.md ("Sub-step transition does not remount the action bar") locked by this key choice.
- **D-03:** Delete the `<LevelEditorActionBar />` mount at `apps/planner/src/features/level-progression/level-progression-row.tsx:243` as part of the same plan to avoid double-render. Plan 12.4-09 component import + driver selectors stay in place — only the mount site moves.
- **D-04:** `[data-testid="advance-to-level-{N+1}"]` testid contract preserved verbatim — Phase 12.4-09 E2E spec `tests/phase-12.4/e2e-humano-guerrero-l1-l16.spec.tsx` continues to select via this attribute. Mount site change is invisible to the spec.

### Skill Increment Gate (R2 — F4)
- **D-05:** New pure module `packages/rules-engine/src/skills/skill-budget.ts` housing `canIncrementSkill(skillId, level, snapshot)` + sibling `nextIncrementCost(skillId, level, snapshot)` helper for clarity. Mirrors Phase 12.3-01 attribute-budget pattern verbatim (`canIncrementAttribute` + `nextIncrementCost` in `packages/rules-engine/src/foundation/ability-budget.ts`).
- **D-06:** Wire predicate at `apps/planner/src/features/skills/skill-sheet.tsx:77` — extend the existing `disabled` expression: `row.disabled || row.currentRank + row.step > row.maxAssignableRank || !canIncrementSkill(row.skillId, level, snapshot)`. Existing per-rank cap (`maxAssignableRank`) preserved; new per-level total-budget cap added as a third `||` condition.
- **D-07:** Snapshot composition uses the existing planner-side `selectSkillBudgetSnapshot` (or equivalent — researcher confirms exact selector name). New module stays framework-agnostic; planner adapter at boundary.

### Scroll Reset (R3 — F2)
- **D-08:** `useLayoutEffect` with `useRef` to the inner scroll container in `apps/planner/src/features/skills/skill-board.tsx` (or `skill-sheet.tsx` per researcher confirmation of the actual scroller host). Reset `scroller.scrollTop = 0` on mount AND on `level` dependency change so navigating between active levels also resets.
- **D-09:** Pre-paint reset (`useLayoutEffect`, not `useEffect`) so user never sees the mid-list flash. Acceptance criterion "no `scrollIntoView` call within 50ms of mount" satisfied by the layout-phase reset firing first.
- **D-10:** Do NOT remove the auto-focus root cause (option G2c rejected) — focus management may serve accessibility; surgical reset is safer than re-architecting focus flow.

### Row Pill Separator (R5 — F1)
- **D-11:** CSS `gap` strategy — add `gap: var(--space-xs)` (or nearest existing token) to the row header flex container in `apps/planner/src/features/level-progression/level-progression-row.tsx` scoped CSS namespace. Visual separation from gap; no text separator characters added.
- **D-12:** Header template prepends literal `Nivel ${level}` token to the row label string — Spanish copy key `shellCopyEs.progression.rowLevelPrefix(level)` added in `apps/planner/src/lib/copy/es.ts` (mirrors the Plan 12.6-01 `shellCopyEs.progression.*` block expansion pattern). Acceptance regex `/Nivel N.+\w+.+\d+\/\d+ dotes.+\d+\/\d+ pts.+[⚠✓✗🔒]/` satisfied because flex `gap` renders as whitespace in `textContent` between adjacent `<span>` children.
- **D-13:** Zero new design tokens, zero new hex colors, zero new `font-weight: 700` — DEF-12.4-02 invariant preserved.
