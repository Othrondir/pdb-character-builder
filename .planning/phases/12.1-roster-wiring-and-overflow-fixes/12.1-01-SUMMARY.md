## Decisions Made

- **Server-rule overlay adapter instead of roster-only projection** — the raw projection dropped `minimumClassCommitment` and `exceptionOverrides`, breaking phase-04 multiclass-rules specs. An id-keyed overlay map preserves these critical rules next to the projection without polluting the extractor-emitted catalog (D-01 "keep non-roster content").
- **Prestige classes keep a synthesized deferred label** — the compiled schema has no deferred-label field, so prestige classes would project as `summaryStatus: 'legal'` at L1, breaking D-02 prestige-gating. Injecting the hand-authored label text (`'Pendiente de dotes o habilidades de fases posteriores.'`) preserves the filter's contract and keeps `tests/phase-04/class-prerequisites.spec.ts:98` green verbatim.
- **Ripple-update phase-04 tests instead of rolling back the TLK label** — the compiled catalog is authoritative per D-01; the test labels were updated to match, not the projection.

- **Found during:** Task 2 (GREEN)
- **Issue:** Plan spec for `projectCompiledClass` emits `minimumClassCommitment: undefined`, `exceptionOverrides: []`, `implementedRequirements: {}`. **2. [Rule 1 - Bug] Prestige classes would render as legal-status at L1 under raw projection**
- **Found during:** Task 2 (GREEN) while re-reading `collectVisibleClassOptions`
- **Issue:** The rules-engine filter's "prestige gating at L1" is encoded in `evaluateClassEntry`, which gates via `deferredRequirementLabels`/`minimumAbilityScores`/`requiresDeity`/`allowedAlignmentIds`. Compiled prestige classes have none of those, so a raw projection would bubble them up with `summaryStatus: 'legal'`, breaking D-02 "preserve the filter".
- **Fix:** Project prestige classes with `deferredRequirementLabels: ['Pendiente de dotes o habilidades de fases posteriores.']` (matches hand-authored shadowdancer/weapon-master pattern). Base classes get `[]` as the plan specified.
- **Files modified:** `apps/planner/src/features/level-progression/class-fixture.ts`
- **Verification:** `tests/phase-12.1/class-roster-wiring.spec.ts` "preserves the prestige-gating filter at L1" passes; `tests/phase-04/class-prerequisites.spec.ts` "keeps shadowdancer blocked while deferred requirements are pending" passes (asserts the label string verbatim).
- **Committed in:** `0e5629d`
