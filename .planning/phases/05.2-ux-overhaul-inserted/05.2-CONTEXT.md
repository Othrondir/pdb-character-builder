# Phase 05.2: UX Overhaul - Context

**Gathered:** 2026-04-15
**Status:** Ready for planning

<domain>
## Phase Boundary

Rebuild the planner's navigation flow, layout structure, visual density, and typography to replicate the NWN1 character creation UI pattern — stepper + list/detail + character sheet — replacing the current web-style vertical scroll of identical cards. This phase restructures existing screens and components; it does not add new game mechanics, data catalogs, or feature capabilities.

</domain>

<decisions>
## Implementation Decisions

### Flow and layout — Stepper NWN1 fiel
- **D-01:** Replace the current vertical scroll of cards with a 3-column layout replicating NWN1: stepper (left) | selection list + detail panel (center) | character sheet (right).
- **D-02:** Each creation step opens its own selection screen in the center area: list of options on the left half, description/detail of the selected item on the right half. Matches the NWN1 pattern exactly (e.g., race selection = race list left + race description right).
- **D-03:** Bottom action bar with Aceptar/Cancelar buttons per step, matching NWN1's button placement.

### Density and hierarchy — Fiel al juego
- **D-04:** Completed steps collapse to a single summary line in the stepper (e.g., "✔ Raza: Humano"). Clicking a completed step reopens it for editing.
- **D-05:** The active step is visually prominent and expanded. Pending steps show as inactive/greyed buttons.
- **D-06:** The right panel is a persistent "Hoja de personaje" with NWN1-style tabs (Stats, Skills, Feats, Spells). Shows dense attribute grid, saving throws, BAB, AC, HP — exactly like the game's character sheet.

### Reference model — NWN1 puro
- **D-07:** NWN2DB is no longer the flow reference. The planner replicates the NWN1 character creation flow and visual identity. NWN2DB is fully discarded as a UX model.
- **D-08:** The planner ADDS the ability to plan 16 levels (which the game doesn't have), but the look, feel, and interaction pattern stays NWN1.

### Visual treatment — Réplica directa NWN1
- **D-09:** Dark background (#0a0a0a / very dark), golden borders (#c8a84e), light text on dark (#e8d5a0 primary, #8b7355 secondary). Discard the current parchment/light background scheme entirely.
- **D-10:** Panel frames with golden/ornamental borders matching the game's UI chrome. Buttons with golden border on dark background.
- **D-11:** The current parchment-and-ink tokens are replaced with a dark-first NWN1 palette. This is a full visual inversion from the current light theme.

### Typography — Ajustar uso, mantener fuentes
- **D-12:** Keep Cormorant Garamond (titles) and Spectral (body) — they are already close to NWN1's serif style.
- **D-13:** Change HOW the fonts are used to match NWN1's GUI feel instead of web display:
  - Title bars: uppercase, letter-spacing 0.15em, regular weight (not light), moderate size (~18px not 36px)
  - Labels and buttons: regular weight, medium size, centered
  - Body/description text: smaller, denser, matching the game's text panels
  - Overall: GUI-density sizing, not web-landing-page sizing
- **D-14:** The visual shift is from "web typography" to "game UI typography" — same fonts, different usage parameters.

### Navigation and structure — Stepper único + rail de niveles
- **D-15:** Eliminate the current 7-section sidebar (Construcción, Habilidades, Conjuros, Atributos, Estadísticas, Resumen, Utilidades). Replace with a single stepper that covers both character creation AND level progression.
- **D-16:** The stepper has two zones:
  - **ORIGEN** section: Raza → Alineamiento → Deidad → Atributos (collapsed when complete)
  - **PROGRESIÓN** section: A compact rail of level buttons [1][2][3]...[16]. Clicking a level expands its sub-steps (Clase, Habilidades, Dotes, Conjuros). Only one level expanded at a time.
- **D-17:** Resumen and Utilidades become special action buttons outside the stepper (bottom of the stepper column or in the header bar), not navigation sections.

### Claude's Discretion
- Exact pixel sizes for the 3-column grid proportions, as long as the stepper is narrow, the center is the widest, and the character sheet is fixed-width
- Exact golden border ornament style (solid line vs beveled vs corner ornaments), as long as it evokes NWN1
- Exact tab icons for the character sheet panel, as long as they represent the same categories as the game
- How to handle mobile/responsive — NWN1 was desktop-only, so the mobile layout is open for interpretation
- Exact animation/transition behavior when switching steps

</decisions>

## Existing Code Insights

### Reusable Assets
- `planner-shell-frame.tsx` — 3-column grid structure can be adapted (sidebar→stepper, content→list/detail, summary→character sheet)
- Zustand stores (character-foundation, level-progression, skills) — state logic is sound, only UI projection changes
- CSS tokens system — token names can be reused, values replaced with dark palette
- Route structure — routes exist for all sections, may need restructuring around stepper model

### Established Patterns
- BEM-style CSS with `.planner-*` namespace — keep this convention
- Zustand + pure selectors for derived state — keep this pattern
- Status modifiers (`.is-blocked`, `.is-illegal`, `.is-selected`) — keep and extend for stepper states

### Integration Points
- Shell frame component is the main restructuring target
- Section nav component gets replaced entirely by stepper
- Summary panel gets replaced by NWN1 character sheet
- Each route's content component needs to adopt the list/detail layout
- CSS tokens are the single point of palette change

</code_context>

<specifics>
## Specific Ideas

- The NWN1 legacy interface screenshots in `legacyinterface/` are the PRIMARY design reference — every layout decision should be checked against these
- The stepper should feel like clicking through the game's creation flow, not like navigating a web app
- Character sheet panel should have the same information density as the game — attribute grid, saving throws, BAB, AC, HP all visible at once without scrolling
- Golden border treatment should match the game's double-line frame style with corner ornaments where feasible
- Typography parameters (uppercase titles with letter-spacing, regular weight, GUI-density sizing) are critical to the NWN1 feel

</specifics>
