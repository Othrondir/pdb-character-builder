# Phase 05.2 Plan 04: Shell Frame Rewrite & Router Simplification Summary

Rewrote the shell frame from route-driven Outlet rendering to store-driven 3-column layout. Connected all Wave 2 components into a working application.

## What Was Done

### Task 1: Rewrite PlannerShellFrame and simplify router

- Replaced the old header+sidebar+outlet shell with a clean 3-column CSS grid: `240px | 1fr | 300px`
- PlannerShellFrame now renders CreationStepper (left), CenterContent (center), CharacterSheet (right)
- Created CenterContent component that reads `activeOriginStep`/`expandedLevel`/`activeLevelSubStep` from shell store and renders the appropriate feature board
- Simplified router from 7 child routes to a single root route with PlannerShellFrame as the component
- Removed old topbar header entirely (NWN1 has no top bar per D-07)
- Added responsive breakpoints for tablet (stepper collapses) and mobile (single column)
- Deleted 8 dead code files (section-nav, section-view, 6 route files) that broke TypeScript compilation

### Task 2: Update Phase 2 tests for new shell structure

- Updated theme-contract.spec.ts to check NWN1 dark tokens and frame/button classes
- Updated layout-shell.spec.ts to verify 3-column layout landmarks (stepper nav, main, character sheet aside)
- Updated navigation-copy.spec.ts to check stepper labels (Raza, Alineamiento, Deidad, Atributos) and section headings
- Updated shell-routes.spec.ts to validate new step registries (originSteps, levelSubSteps, sheetTabs)
- All tests render PlannerShellFrame directly instead of through RouterProvider to avoid Transitioner jsdom bug

## Deviations

1. **Dead route files deleted early** (Rule 3 — auto-fixed): Plan said to leave dead route files for Plan 05 cleanup, but they caused TypeScript compilation failures since sections.ts no longer exports PlannerSectionId/plannerSections. Deleted them in Task 1 instead. No impact on Plan 05 scope since it focuses on feature board adaptation.

2. **Tests render without router** (Rule 3 — auto-fixed): Plan specified tests using `createPlannerRouter` + `RouterProvider`, but TanStack Router Transitioner has a jsdom compatibility bug (`router.stores.status.set is not a function`). Phase 05.2 tests already render components directly — matched that pattern for consistency.
