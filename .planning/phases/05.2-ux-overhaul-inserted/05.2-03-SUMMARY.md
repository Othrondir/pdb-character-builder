# Phase 05.2 Plan 03: Center Content Primitives and Character Sheet Summary

Six NWN1 layout components providing the list+detail center content pattern and right-column tabbed character sheet with real attribute data from existing selectors.

## One-liner

SelectionScreen provides the universal NWN1 list+detail split; CharacterSheet renders tabbed stats from foundation and progression selectors with empty state handling.

## What Was Built

### Task 1: Center Content Primitives

- **SelectionScreen**: Wraps NwnFrame as a section with title bar, two-column grid content area, and optional action bar slot. This is the universal center content wrapper for every creation step (race, alignment, class, skills, feats, spells).
- **OptionList**: Scrollable listbox with role="option" items. Supports selected, blocked, disabled states with ARIA attributes. Accepts optional renderItem override for custom row rendering.
- **DetailPanel**: Right-half panel with optional title (h3), body text, and custom children slot for the detail/description area of each selection step.
- **ConfirmDialog**: Native HTML `<dialog>` element with showModal/close lifecycle. Uses nwn-frame styling for the modal border treatment. Spanish copy: Cancelar/Aceptar buttons.

### Task 2: Character Sheet Right Column

- **SheetTabs**: ARIA tablist with 4 tabs (Estadisticas, Habilidades, Dotes, Conjuros) driven by shell store `characterSheetTab` state. Active tab gets gold bottom border indicator.
- **CharacterSheet**: Right-column aside replacing the old SummaryPanel pattern. Renders:
  - Title bar: "Hoja de personaje"
  - SheetTabs navigation
  - Stats panel: 6 ability scores (Fuerza, Destreza, Constitucion, Inteligencia, Sabiduria, Carisma) with numeric values and modifier display. Derived stats (CA, PG, BAB, Fortaleza, Reflejos, Voluntad) from foundation and progression selectors.
  - Placeholder panels for Skills, Feats, Spells tabs
  - Empty state when no race selected: "La hoja aun esta vacia"

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] NwnFrame extended for aria-label pass-through**
- **Found during:** Task 2
- **Issue:** CharacterSheet requires `aria-label="Hoja de personaje"` on the aside element, but NwnFrame only accepted `as`, `children`, `className` props -- no mechanism to pass HTML attributes.
- **Fix:** Extended NwnFrame interface with `Omit<ComponentPropsWithoutRef<'div'>, 'children' | 'className'>` spread, passing `...rest` to the rendered tag.
- **Files modified:** apps/planner/src/components/ui/nwn-frame.tsx
- **Commit:** a3ebf44

**2. [Rule 3 - Blocking] Stepper copy and SheetTab types added ahead of Plan 02 merge**
- **Found during:** Task 2
- **Issue:** Plan 03 references `shellCopyEs.stepper.sheetTabs` and `SheetTab` type from sections.ts, but these are created by Plan 02 (running in parallel). Without them, SheetTabs and CharacterSheet cannot compile.
- **Fix:** Added `stepper` copy block to es.ts, `SheetTab` type and `sheetTabs` array to sections.ts, and `characterSheetTab`/`setCharacterSheetTab` to planner-shell.ts as additive extensions. Plan 02 will rewrite these files entirely at merge time; the additions are compatible with either pre- or post-merge state.
- **Files modified:** es.ts, sections.ts, planner-shell.ts
- **Commit:** a3ebf44
