### `components/shell/creation-stepper.tsx` (component, event-driven)

**Analog:** `components/shell/section-nav.tsx`

**Imports pattern** (section-nav.tsx lines 1-14):
```typescript

import type { PlannerSectionId } from '@planner/lib/sections';  // BECOMES step type imports

```
Icons will come from `lucide-react` (already installed) -- same pattern as section-nav.tsx lines 1-9.

**Navigation container pattern** (section-nav.tsx lines 36-70):
```tsx
<nav
  aria-label="Navegaci\u00f3n principal del planificador"
  className="planner-nav planner-panel planner-panel--nav"
>
  <ul className="planner-nav__list">
    {plannerSections.map((section) => {
      const isActive = section.id === currentSectionId;
      return (
        <li key={section.id}>
          <Link
            aria-current={isActive ? 'page' : undefined}
            className={`planner-nav__link${isActive ? ' is-active' : ''}`}
            onClick={() => setActiveSection(section.id)}
            to={section.path}
          >
            {/* ... */}
          </Link>
        </li>
      );
    })}
  </ul>
</nav>
```
**Adaptation:** Replace `<ul>` + `<Link>` with `<ol>` + `<button>`. Use `aria-current="step"` instead of `aria-current="page"`. Section list becomes two groups: ORIGEN steps (ordered list) and PROGRESION level rail (4x4 grid). Step click dispatches to shell store `setActiveStep()` instead of navigating a route.

**Store interaction pattern** (section-nav.tsx lines 31-34):
```typescript
const setActiveSection = usePlannerShellStore((state) => state.setActiveSection);
```
Becomes:
```typescript
const setActiveStep = usePlannerShellStore((state) => state.setActiveStep);
const activeStep = usePlannerShellStore((state) => state.activeStep);
const expandedLevel = usePlannerShellStore((state) => state.expandedLevel);
```

**Step completion derivation pattern** (from RESEARCH.md Pitfall 6):
Step completion must be DERIVED from domain stores, not stored in shell. Read existing selectors:
```typescript
import { selectOriginReadyForAbilities } from '@planner/features/character-foundation/selectors';
import { selectLevelRail } from '@planner/features/level-progression/selectors';
```

### `ui/confirm-dialog.tsx` (component, modal)

**No analog in codebase.** Use native HTML `<dialog>` per RESEARCH.md recommendation.

**Pattern from RESEARCH.md:**
```tsx
export function ConfirmDialog({ open, onConfirm, onCancel, title, body }: ConfirmDialogProps) {
  const dialogRef = useRef<HTMLDialogElement>(null);

  useEffect(() => {
    if (open) dialogRef.current?.showModal();
    else dialogRef.current?.close();
  }, [open]);

  return (
    <dialog ref={dialogRef} className="nwn-frame confirm-dialog">
      <h2>{title}</h2>
      <p>{body}</p>
      <div className="confirm-dialog__actions">
        <NwnButton variant="secondary" onClick={onCancel}>Cancelar</NwnButton>
        <NwnButton variant="primary" onClick={onConfirm}>Confirmar</NwnButton>
      </div>
    </dialog>
  );
}
```
