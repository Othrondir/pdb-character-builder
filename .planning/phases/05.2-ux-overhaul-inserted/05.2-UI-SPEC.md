## Spacing Scale

Declared values (multiples of 4). Carried from existing `tokens.css` unchanged:

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Icon gaps, inline padding, tight label gaps |
| `--space-sm` | 8px | Compact element spacing, stepper entry gaps, list item gaps |
| `--space-md` | 16px | Default element spacing, inner panel padding |
| `--space-lg` | 24px | Panel section padding, group separators |
| `--space-xl` | 32px | Layout column gaps, major section breaks |
| `--space-2xl` | 48px | Page-level vertical spacing (header to body) |
| `--space-3xl` | 64px | Reserved; not expected in this phase |

Exceptions:
- Stepper buttons: 40px min-height (touch target, matches existing attribute control buttons)
- Level rail buttons: 36px min-dimension (compact square buttons for [1][2]...[16])
- Action bar buttons: 44px min-height (Aceptar/Cancelar, matches NWN1 button proportion)

Source: existing `tokens.css`, NWN1 screenshots

## Color

Full visual inversion from current light theme. New dark-first palette:

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--color-surface` | `#0a0a0a` | Page background, body fill |
| Secondary (30%) | `--color-panel` | `#141010` | Panel interiors, stepper background, card backgrounds |
| Panel edge | `--color-panel-edge` | `rgba(200, 168, 78, 0.35)` | Inner frame borders (double-line effect) |
| Accent (10%) | `--color-gold` | `#c8a84e` | Panel frame borders, active step indicator, selected item highlight border, CTA button borders, stepper completed checkmark |
| Primary text | `--color-text` | `#e8d5a0` | Body text, headings, stat labels |
| Secondary text | `--color-text-muted` | `#8b7355` | Inactive stepper steps, meta labels, disabled descriptions |
| Selection fill | `--color-selection` | `rgba(200, 168, 78, 0.18)` | Active list item background, hovered stepper step |
| Selection border | `--color-selection-border` | `rgba(200, 168, 78, 0.75)` | Selected item border, active step border |
| Destructive | `--color-destructive` | `#7a1f1f` | Illegal validation states, error borders |
| Blocked | `--color-blocked` | `rgba(200, 168, 78, 0.55)` | Blocked validation states, warning borders |

Accent (`--color-gold` / `#c8a84e`) reserved for:
1. Panel frame borders (outer and inner double-line)
2. Active stepper step indicator and border
3. Selected list item border
4. Aceptar button border
5. Completed step checkmark icon
6. Character sheet tab active indicator
7. Section title bar bottom border accent line

Elements that do NOT use accent:
- Body text (uses `--color-text`)
- Inactive buttons (uses `--color-text-muted` border)
- Panel interior fills (uses `--color-panel`)
- Cancelar button (uses `--color-text-muted` border, not gold)

Source: CONTEXT.md D-09, D-10, D-11, NWN1 screenshots

## Panel Frame Treatment

NWN1 panels use a distinctive double-border frame with corner ornaments. Web replica:

```
Outer border: 2px solid var(--color-gold)
Inner border: 1px solid var(--color-panel-edge)  (inset 6px via ::before pseudo-element)
Corner ornaments: CSS border-image or pseudo-element notches at four corners
Background: var(--color-panel)
Border radius: 0px (NWN1 uses sharp corners, not rounded)
```

This replaces the current `border-radius: 22px` rounded panels with sharp-cornered NWN1 frames.

Panel types:
- **Frame panel** (stepper, character sheet, content area): Double-border, sharp corners, dark fill
- **List item** (race option, class option, skill row): Single gold border, dark fill, 0px radius
- **Button** (Aceptar, Cancelar, stepper step): Single gold border, dark fill, 0px radius

Source: NWN1 screenshots (all panels show sharp rectangular frames with double borders), CONTEXT.md D-10

## Stepper Interaction Model

### ORIGEN Section

| State | Visual | Interaction |
|-------|--------|-------------|
| Pending (locked) | `--color-text-muted` text, muted border | Not clickable |
| Active (current) | `--color-selection` fill, gold border, bright text | Content loads in center |
| Complete | Gold text, checkmark icon, summary line (e.g. "Humano, Caotico bueno") | Click to reopen for editing |
| Blocked | `--color-blocked` border, warning icon | Click shows validation message |

Steps in order: Raza, Alineamiento, Deidad, Atributos.
Each step unlocks only after the previous is completed (existing behavior from Phase 3).

### PROGRESION Section

| State | Visual | Interaction |
|-------|--------|-------------|
| Empty level | Muted number, muted border | Click to start configuring |
| Active level | Gold fill, bright number | Sub-steps visible below |
| Configured level | Gold border, bright number, class abbreviation below | Click to expand |
| Blocked level | `--color-blocked` border, warning icon overlay | Click shows dependency |
| Illegal level | `--color-destructive` border, error icon overlay | Click shows violation |

Level rail layout: Compact grid, 4 columns of level buttons (4x4 = 16 levels).
Only one level expanded at a time.

Expanded level sub-steps: Clase, Habilidades, Dotes, Conjuros (vertical list below the level button).

Source: CONTEXT.md D-04, D-05, D-08, D-16

## Animation and Transitions

| Transition | Duration | Easing | Property |
|------------|----------|--------|----------|
| Step change (center content swap) | 200ms | ease-out | opacity, transform translateY(8px) |
| Panel hover | 150ms | ease | background-color, border-color |
| Stepper expand/collapse | 250ms | ease-in-out | max-height, opacity |
| Level sub-steps reveal | 200ms | ease-out | max-height, opacity |
| Character sheet tab switch | 150ms | ease | opacity |

No spring animations. No parallax. Transitions should feel immediate and mechanical like the game UI, not fluid like modern web.

## Responsive Behavior

NWN1 was a desktop-only game. The planner is desktop-first. Mobile is a graceful degradation, not a primary target.

| Breakpoint | Layout |
|------------|--------|
| >= 1024px | Full 3-column layout as specified |
| 768px - 1023px | 2-column: stepper collapses to horizontal bar at top, center + character sheet side by side |
| < 768px | Single column: hamburger menu for stepper, character sheet as collapsible bottom drawer |

At < 1024px the Aceptar/Cancelar action bar becomes a sticky bottom bar.

Source: Claude's discretion (CONTEXT.md allows mobile interpretation)

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Color contrast | `#e8d5a0` on `#0a0a0a` = 11.7:1 (AAA). `#8b7355` on `#0a0a0a` = 4.6:1 (AA). `#c8a84e` on `#0a0a0a` = 7.2:1 (AAA). |
| Focus indicator | 2px solid `--color-gold`, 2px offset on all interactive elements |
| Stepper semantics | `<nav>` with `<ol>` for ordered steps, `aria-current="step"` on active |
| Level rail semantics | `<div role="radiogroup">` with `<button role="radio">` per level |
| List selection | `aria-selected` on selected list item, `role="listbox"` on list |
| Character sheet tabs | `role="tablist"` + `role="tab"` + `role="tabpanel"` |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables all transitions |
| Screen reader | `.sr-only` class already exists, use for icon-only buttons |
