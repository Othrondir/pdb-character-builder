**origin-board.tsx -- rewrite to use SelectionScreen pattern (per D-02):**

Current structure: The OriginBoard renders all 4 origin steps (race, subrace, alignment, deity) as vertical card sections on a single page with an `OriginSheet` aside. This becomes a single-step view driven by the `activeStep` prop.

New structure: OriginBoard accepts an `activeStep: 'race' | 'alignment' | 'deity'` prop from center-content.tsx. It renders a `SelectionScreen` showing the option list for that specific step.

```tsx
import { SelectionScreen } from '@planner/components/ui/selection-screen';
import { OptionList } from '@planner/components/ui/option-list';
import { DetailPanel } from '@planner/components/ui/detail-panel';
import { ActionBar } from '@planner/components/ui/action-bar';
import { useCharacterFoundationStore } from './store';
import { selectOriginOptions } from './selectors';
import { usePlannerShellStore } from '@planner/state/planner-shell';
import { shellCopyEs } from '@planner/lib/copy/es';
import type { OriginStep } from '@planner/lib/sections';

interface OriginBoardProps {
  activeStep: 'race' | 'alignment' | 'deity';
}
```

For `activeStep === 'race'`:
- Title: "Selecciona la raza de tu personaje" (add to es.ts if not present)
- OptionList items: map `originOptions.races` to `OptionItem[]` (id, label, selected, blocked)
- DetailPanel: show the selected race's description from fixture data. If no description is available in the fixture, show a generic "Selecciona una raza para ver su descripcion." message.
- ActionBar: Aceptar advances to alignment step (`setActiveOriginStep('alignment')`), disabled if no race selected. Cancelar clears race selection.

For `activeStep === 'alignment'`:
- Title: "Selecciona el alineamiento del personaje"
- OptionList items: map `originOptions.alignments` to `OptionItem[]`. The alignment grid (3x3) can use the OptionList with a custom CSS class for grid layout, OR use a custom render. Use a CSS grid override: `option-list--alignment-grid` class with `display: grid; grid-template-columns: repeat(3, 1fr);` applied to the option-list div.
- ActionBar: Aceptar advances to deity step.

For `activeStep === 'deity'`:
- Title: "Selecciona la deidad del personaje"
- OptionList items: map `originOptions.deities`
- ActionBar: Aceptar advances to attributes step.

The `onSelect` handlers call existing store actions: `setRaceId`, `setAlignmentId`, `setDeityId`. These are already on `useCharacterFoundationStore`.

Keep all existing selector usage (`selectOriginOptions`). Only the visual projection changes -- store logic is untouched.

Remove the old inline `OriginOptionButton`, `OriginStep` (the local component), and `OriginSheet` components. Remove imports of old `planner-chip`, `planner-section-view` class references.

Add to es.ts the step-specific screen titles:
