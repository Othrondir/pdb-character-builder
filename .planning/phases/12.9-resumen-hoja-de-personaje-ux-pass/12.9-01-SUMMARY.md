## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking issue] Added runtime guard `(model.attributes ?? [])` alongside the `@ts-expect-error` breadcrumb.**
- **Found during:** Task 1 verification (phase-08 vitest run after the selector shape change).
- **Issue:** The plan's "must truths" and cross-plan contract describe the `@ts-expect-error` breadcrumb as a typecheck-only seam that preserves runtime behavior. In practice, `ResumenTable` is mounted by several Phase 8 specs (`json-import-mismatch.spec.tsx`, `share-fallback.spec.tsx`, etc.) through the shell render path, and `model.attributes.map` on `undefined` throws `TypeError: Cannot read properties of undefined (reading 'map')` at the first render. 7 otherwise-green phase-08 specs regressed.
- **Fix:** Wrapped the call with `(model.attributes ?? [])` — the `@ts-expect-error` is still consumed because `.attributes` on `ResumenViewModel` is still a missing property (the property access itself, not the `.map` call, is what TypeScript flags). Runtime now renders an empty `<tbody>` until Plan 02 deletes the block entirely.
- **Files modified:** `apps/planner/src/features/summary/resumen-table.tsx` (1 line changed).
- **Commit:** `5a803be`.
- **Plan 02 impact:** none — Plan 02 Task 1 rewrites the file end-to-end; the `??` guard disappears alongside the block and the breadcrumb.

**2. [Rule 1 - Obsolete coverage] Deleted two test cases in `tests/phase-08/resumen-selectors.spec.ts` that asserted against the deleted `attributes[]` field.**
- **Found during:** Task 1 verification (phase-08 vitest run after the runtime guard above).
- **Issue:** `it('emits six attribute rows in STR/DEX/CON/INT/WIS/CHA order', ...)` and `it('derives ability modifier as floor((total - 10) / 2)', ...)` both read `result.current.attributes` which is now intentionally undefined per D-08. The threat register's T-12.9-01-05 disposition (`accept`) reasoned that `ResumenViewModel.attributes` is not referenced by `@planner/features/persistence/**` — true, but missed this selector-level regression spec. The plan's acceptance criterion `pnpm exec vitest run tests/phase-08/ exits 0` conflicts with the threat disposition.
- **Fix:** Deleted the two now-obsolete cases; replaced with an explanatory comment block referencing Plan 12.9-01 / D-08. Preserved four other cases (identity, 20-level progression, cumulativeBab, skill-row coverage) that still exercise the surviving selector contract.
- **Files modified:** `tests/phase-08/resumen-selectors.spec.ts` (22 lines deleted, 6 lines added).
