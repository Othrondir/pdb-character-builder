## Top 3 Priority Fixes

1. **Action bar trails a phantom 6th flex cell in empty-state.** The hidden `<input type="file">` inside `.resumen-board__actions` participates in flex sizing — visible at 1440px+ as a large empty gold-bordered slot next to Compartir. *User impact:* visually implies a 6th missing action, wastes ~18% of the action-bar width. *Concrete fix:* add `.resumen-board__actions > input[type="file"] { display: none; }` (or move the hidden input outside the `<NwnFrame>` JSX child list while keeping the ref).

2. **Empty-state action buttons render as oversized empty boxes before race selection.** At 1440px viewport, the 5 action buttons + phantom input cell stretch to ~200px tall each because the flex container has no `align-items: flex-start` and the buttons inherit full-height stretching when the row contains no tall sibling. *User impact:* creates visual noise before the user has done anything; undercuts the empty-state's "come back later" voice. *Concrete fix:* add `.resumen-board__actions { align-items: flex-start; justify-content: flex-start; }` (or `center`) to pin buttons to intrinsic height.

3. **Identity header has no semantic heading landmark.** The new `<header className="resumen-table__identity-header">` renders 4 pieces separated by middots but contains NO `<h3>` / `<h4>` / `role="heading"` — the old `<h3 id="resumen-identity-heading">` was deleted per D-03. *User impact:* screen-reader users lose the identity waypoint when navigating Resumen via `h` key; heading hierarchy now jumps from board `<h2>` directly to block `<h3>`s, skipping identity. *Concrete fix:* wrap the character name in `<h3 className="sr-only">` OR apply `aria-label="Identidad del personaje"` to the `<header>` element; keep current visual treatment unchanged.
