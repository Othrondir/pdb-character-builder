# Phase 12.9 — UI Design Contract

> Visual and interaction contract for the `<ResumenBoard>` / `<ResumenTable>` UX pass.
> Generated by gsd-ui-researcher from locked CONTEXT decisions D-01..D-10 + SPEC.md R1..R6.
> Verified by gsd-ui-checker; consumed by gsd-planner + gsd-executor.

---

## Scope Recap

This is a **scoped visual polish pass** over a single route (`<ResumenBoard>` at `apps/planner/src/features/summary/`). It:

- Fixes Progresión 8-col overlap via full-width grid breakout (R1 / D-01).
- Deletes the identity `<dl>` + the attributes `<table>` in favor of a single compact `<header>` line (R2 / D-03).
- Adopts the Phase 12.8 Dotes 8px inner-padding baseline on `.resumen-table__heading`, `.resumen-table__block`, `.resumen-board__actions` (R3 / D-05).
- Adds a dedicated pre-creation empty-state frame (R4 / D-04).
- Keeps the Save / Load / Export / Import / Share dialog contract byte-identical (R5 / D-07).
- Deletes `ResumenViewModel.attributes` when no other consumer remains (R6 / D-08).

**Out of scope (locked by SPEC.md Boundaries):** right-rail aside `<CharacterSheet>`, persistence helpers, rules-engine, extractor, share-URL arithmetic, new CSS tokens, new hex colors, new `font-weight: 700`, mobile <1024px.

## Layout & Interaction Contracts

### L-1: Progresión full-width grid breakout (R1 / D-01)

- Selector: `.resumen-table__block--progresion` (BEM modifier on the Progresión `<section>`; planner may choose `--progression` as long as BEM consistency is preserved — D-01 + Discretion note).
- Declaration: `grid-column: 1 / -1;` — breaks the block out of the parent `.resumen-table` `repeat(auto-fit, minmax(18rem, 1fr))` grid to span every track.
- The parent `.resumen-table` grid rule **must remain unchanged** (identity-header + Habilidades continue sharing auto-fit tracks per D-02).
- Interaction contract: at viewport ≥ 1024px, every `<th>` + `<td>` of `.resumen-table__progression` is visible without horizontal scroll. Automated assertion: `scrollWidth <= clientWidth` at 1280px test viewport (D-10).
- No column merges, no column drops, no header abbreviations beyond the existing `Fort.` / `Ref.` / `Vol.` shortcuts.

### L-2: Habilidades full-width grid breakout (D-02)

- Selector: `.resumen-table__block--habilidades` (BEM mirror of L-1).
- Declaration: `grid-column: 1 / -1;` — same mechanism as L-1.
- Rationale: 39 skills × 4 cols cramps an 18rem auto-fit track; full-width simplifies scan rhythm (identity-header compact on top, then Progresión full-width, then Habilidades full-width).
- No CSS `columns: 2` multi-column trick (rejected per D-02 — breaks `<th>` / `<td>` row associations across column breaks, accessibility regression).

### L-3: Identity header replacement (R2 / D-03)

- **Deleted elements:**
  - `<dl class="resumen-table__identity">` (4 dt/dd pairs: Nombre / Raza / Alineamiento / Ruleset · Dataset).
  - `<table class="resumen-table__attrs">` (6 attribute rows × Total × Mod columns).
  - `<h3 id="resumen-identity-heading">` — block heading for the old Identidad block.
  - Associated CSS: `.resumen-table__identity`, `.resumen-table__attrs`, `.resumen-table__identity dt`, `.resumen-table__identity dd` (all deleted at app.css:2031-2038).
- **Added element:** single `<header class="resumen-table__identity-header">` rendering:
  ```
  {name} · {race}{subrace ? ` · ${subrace}` : ''} · {alignment} · <span>{dataset}</span>
  ```
  with the `<span>` scoped to the dataset text only and receiving `color: var(--color-gold-faded)`.
- **Placement (Discretion — planner choose):**
  - Option A: As the first child of the existing `.resumen-table` grid, BEFORE the Progresión block (no `.nwn-frame` wrapper — plain compact header above framed blocks).
  - Option B: Inside its own `.resumen-table__block.nwn-frame` with `grid-column: 1 / -1` (full-width framed header).
  - **Recommendation:** Option A. Rationale: the ficha hand-off surface reads cleaner with a compact unframed identity line sitting above framed data blocks, mirroring the xlsx reference; also saves vertical space at the top of the route.
- No `<dt>`/`<dd>` pairs, no "Nombre:" / "Raza:" prefix labels — the stepper summary bar already provides labeled identity (R2 dedup rationale).
- Grep gate: `grep -c resumen-table__attrs apps/planner/src/features/summary/resumen-table.tsx` returns `0`; `grep -c resumen-table__identity-header apps/planner/src/features/summary/resumen-table.tsx` returns `1`.

### L-4: Empty-state frame (R4 / D-04)

- Short-circuit in `<ResumenBoard>`: **BEFORE** calling any selector, check `isProjectable = raceId !== null && alignmentId !== null` (predicate already exists at `resumen-board.tsx:41`).
- Behavior:
  - `isProjectable === false` → render `<header>.resumen-board__header` + `<NwnFrame>.resumen-board__actions` (unchanged — gates handle disabled state) + `<NwnFrame className="resumen-board__empty-state">` with the new copy — **NO** `<ResumenTable>`.
  - `isProjectable === true` → render the full existing tree including `<ResumenTable>`.
- Empty-state frame structure:
  ```tsx
  <NwnFrame className="resumen-board__empty-state">
    <h3>{copy.emptyState.heading}</h3>
    <p>{copy.emptyState.body}</p>
  </NwnFrame>
  ```
- **No CTA button** (D-04 explicit). The stepper on the left rail always provides navigation back to Construcción; adding a button would require `usePlannerShellStore.setActiveView('construccion')` plumbing + `activeLevel`/`expandedLevel` reset — rejected per CONTEXT Deferred Ideas.
- Action bar (`Cargar` + `Importar`) must remain clickable in empty state so the user can load a saved build into the empty slot. `Save` / `Export` / `Share` stay disabled with existing `title={shellCopyEs.persistence.incompleteBuild}` tooltip.
- CSS: new `.resumen-board__empty-state` selector inherits `.nwn-frame` body via the className combination; adds `padding: var(--space-md, 16px)` inner padding and typographic defaults matching `.resumen-table__block` rhythm. Heading uses `.resumen-table__heading` styling conventions (display font, gold tint) — may reuse the selector or author a scoped variant.

### L-5: Inner-padding baseline adoption (R3 / D-05)

- Per D-05 / Dotes 8px baseline (see `Spacing Scale → Concrete padding deltas` table above).
- `.resumen-table__heading` adds `padding: 0 var(--space-sm, 8px)` — matches `.feat-board__section-header`.
- `.resumen-board__actions` changes from `padding: 0.75rem 1rem` → `padding: var(--space-sm, 8px) var(--space-sm, 8px)` (symmetric 8px; removes asymmetric 12/16 → adopts 8/8).
- `.resumen-table__block` keeps its outer padding (`--space-md` = 16px), semantically renamed from `1rem` literal to `var(--space-md, 16px)`.

### L-6: Action bar preservation (R5 / D-07)

- `<NwnFrame className="resumen-board__actions">` renders **exactly five** `<NwnButton>` children in this fixed order: Guardar → Cargar → Exportar JSON → Importar JSON → Compartir.
- Each button's `disabled` / `onClick` / `title` / `variant` assignment is byte-identical to current master.
- Hidden `<input type="file" accept="application/json" />` remains inside `.resumen-board__actions` (no visual change).
- Dialog mounts (`<SaveSlotDialog>` / `<LoadSlotDialog>` / `<VersionMismatchDialog>`) remain at the current JSX positions.
- Grep gate: `git diff apps/planner/src/features/persistence/` + `git diff apps/planner/src/features/summary/save-slot-dialog.tsx` return empty.

### L-7: Keyboard + focus (accessibility)

- `<header class="resumen-table__identity-header">` is a semantic landmark — no explicit `role` needed (native `<header>` is implicit `banner` when top-level, but scoped under `<section>` it's neutral).
- Block `<section>` elements retain their existing `aria-labelledby` wiring to their `<h3>` headings (Progresión + Habilidades). Identity block loses its heading with the R2 collapse — the `<section>` becomes a plain `<header>` element, no `aria-labelledby` needed.
- Empty-state `<NwnFrame>` uses `<h3>` for heading (assertive landmark position); no `aria-live` needed (empty state is cold start, not a dynamic notification).
- Action buttons inherit `NwnButton` native focus treatment (gold outline on `:focus-visible` — existing CSS).
- Tab order preserved: board heading → action bar (5 buttons) → Progresión table → Habilidades table (or empty-state frame).

### L-8: Responsive behavior

- Phase 12.9 locks the acceptance criteria at viewport ≥ 1024px only (SPEC Boundaries).
- At <1024px, existing mobile behavior ships unchanged — the auto-fit grid with `minmax(18rem, 1fr)` already collapses to single-column on narrow screens; the `grid-column: 1 / -1` modifiers from L-1 + L-2 are no-ops in single-column mode.
- Deferred: dedicated mobile/narrow-viewport Resumen redesign (CONTEXT Deferred Ideas).
