# Phase 12.9: Resumen (Hoja de personaje) UX Pass — Specification

**Created:** 2026-04-24
**Ambiguity score:** 0.18 (gate: ≤ 0.20)
**Requirements:** 6 locked

## Goal

The `<ResumenBoard>` route renders a cleanly-laid, non-duplicative, non-overlapping character ficha at every configured build state (empty → Elfo + Neutral puro + Guerrero L1..L20), with spacing that matches the Phase 12.8 Dotes 8px inner-padding baseline and with Save / Load / Export / Import / Share dialog surfaces preserved functionally verbatim.

## Background

`<ResumenBoard>` (`apps/planner/src/features/summary/resumen-board.tsx`) activates via the stepper "Resumen" button — `center-content.tsx:17` swaps `BuildProgressionBoard` for `ResumenBoard` when `activeView === 'resumen'`. It renders:

1. `<header>.resumen-board__header` — H2 heading + dataset small-caps label.
2. `<NwnFrame>.resumen-board__actions` — 5 NwnButtons (Guardar / Cargar / Exportar JSON / Importar JSON / Compartir), gated by `isProjectable = raceId !== null && alignmentId !== null`.
3. `<ResumenTable>` — three `.resumen-table__block.nwn-frame` sections laid out in a single CSS grid (`.resumen-table`, `grid-template-columns: repeat(auto-fit, minmax(18rem, 1fr))` at `app.css:2009-2013`):
   - **Identidad y atributos** — `<dl>` (Nombre / Raza / Alineamiento / Ruleset · Dataset) + 3-col attributes table.
   - **Progresión** — 20-row × 8-col table (Nivel / Clase / BAB / Fort. / Ref. / Vol. / Dote general / Dote de clase).
   - **Habilidades** — 39-row × 4-col table (Habilidad / Rangos / Mod. atr. / Total).

UAT-2026-04-24 live re-sweep flagged the Resumen route as visually broken:

- Progresión block overlaps (root cause: 8-col table squeezed into an ~18rem auto-fit grid column at default viewport).
- Identidad + atributos block duplicates information already surfaced by the creation-stepper summary bar.
- Spacing inside `.resumen-table__heading` + `.resumen-board__actions` does not inherit the 8px inner-padding baseline established by Phase 12.8 E11/E13/E15 (`.feat-picker__partition-heading`, `.feat-board__section-header`, `.feat-picker__counter`).
- Pre-race/alignment state prints `Sin nombre` + 20 dash rows + 39 dash rows — no dedicated empty surface.

Orthogonal note: the right-rail aside `<CharacterSheet>` (`character-sheet.tsx`) shares the "Hoja de personaje" label but is a separate component; this phase does NOT touch it.

## Requirements

1. **R1 — Progresión full-width breakout**: The 20-row × 8-col Progresión block renders every column inline at the default dev viewport without horizontal scroll.
   - Current: `.resumen-table` is a `repeat(auto-fit, minmax(18rem, 1fr))` grid, and the Progresión block shares one of those auto-fit columns. At `localhost:5173` default viewport (≈1280px minus shell side-rails), the 8-col table overflows its ~18rem column and visually overlaps adjacent blocks / own content.
   - Target: Progresión block breaks out of the auto-fit grid and spans the full content width of `.resumen-table` (`grid-column: 1 / -1` or equivalent), leaving identidad-header + Habilidades free to share the remaining grid tracks.
   - Acceptance: At viewport width ≥ 1024px, every `<th>` and `<td>` of `.resumen-table__progression` is visible without triggering the block's horizontal scrollbar; automated spec asserts `.resumen-table__progression` `scrollWidth <= clientWidth` at a 1280px test viewport; live MCP Chrome UAT against Elfo + Neutral puro + Guerrero L1..L20 confirms no column truncation in 20/20 rows.

2. **R2 — Identidad dedup**: The Resumen block no longer re-prints identity / attribute totals that the creation-stepper summary bar already renders.
   - Current: `<ResumenTable>` renders `<dl>.resumen-table__identity` (Nombre / Raza / Alineamiento / Ruleset · Dataset) AND `<table>.resumen-table__attrs` (six attributes × total × modifier). The stepper summary bar already carries Nombre / Raza / Alineamiento + attribute chips.
   - Target: The identity block collapses to a single compact header line `{name} · {race}{subrace?} · {alignment} · {dataset}` rendered INSIDE `<ResumenTable>` (preserving ficha-surface self-containment for hand-off). The attributes `<table>.resumen-table__attrs` is deleted from `<ResumenTable>`; atributos surfaces only through the stepper summary bar.
   - Acceptance: Grep `apps/planner/src/features/summary/resumen-table.tsx` returns `0` matches for `resumen-table__attrs`; the rendered DOM of `<ResumenTable>` contains exactly `1` `.resumen-table__identity-header` element with the four pieces comma-or-middot separated; `ResumenViewModel.attributes` is no longer consumed by `<ResumenTable>` (TypeScript compile confirms if the field is deleted or stays unused — see R6 for selector policy).

3. **R3 — Spacing inherits Dotes 8px baseline**: `.resumen-board__actions`, `.resumen-table__block`, and `.resumen-table__heading` adopt the symmetric 8px horizontal inner-padding rhythm established by Phase 12.8 E11/E13/E15.
   - Current: `.resumen-board__actions` has `padding: 0.75rem 1rem` (12px vertical / 16px horizontal); `.resumen-table__block` has `padding: 1rem` (16px). `.resumen-table__heading` has no explicit inner-side padding beyond the block's 16px. Inconsistent with the Dotes `.feat-picker__partition-heading` / `.feat-board__section-header` 8px horizontal baseline.
   - Target: Inner padding of `.resumen-table__block` + headings align with the 8px token (`var(--space-sm)` where defined; literal `0.5rem` acceptable when an existing token is not introduced). `.resumen-board__actions` inner padding matches the Dotes section-header inner padding.
   - Acceptance: `git diff apps/planner/src/styles/app.css` for the `.resumen-table__*` and `.resumen-board__actions` selectors shows padding values aligned to the Dotes baseline; no new CSS tokens are introduced; no new hex colors; no new `font-weight: 700` declarations (Phase 12.8 invariants preserved); visual re-UAT at the locked fixture confirms headings and action-row inner spacing are consistent with Dotes `.feat-board__section-header`.

4. **R4 — Empty-state surface**: Pre-race-or-alignment, the Resumen route renders a single explanatory frame instead of the zeroed ficha.
   - Current: When `raceId === null || alignmentId === null`, `<ResumenTable>` still renders (all rows default to `emptyNamePlaceholder = 'Sin nombre'` + `notAvailable = '—'`). The action bar hides Save / Export / Share (disabled) but Cargar / Importar stay enabled; the table body below still paints 20 dashed progression rows + 39 dashed skill rows.
   - Target: When `!isProjectable` (same predicate already used by the action bar), `<ResumenTable>` is replaced by a single `.resumen-board__empty-state.nwn-frame` that copies the stepper empty-state voice (Spanish-first, points user back to Construcción / Atributos). Action bar stays mounted with its existing gating (Cargar + Importar remain usable so the user can load a saved build into the empty slot).
   - Acceptance: With cleared localStorage + IndexedDB and no race picked, the rendered DOM contains `0` `.resumen-table__progression` elements and exactly `1` `.resumen-board__empty-state` element; copy string is added to `shellCopyEs.resumen.emptyState.*` (heading + body) and is not the stepper's `stepper.emptySheetHeading` reused verbatim (ficha voice, not stepper voice); after selecting a race + alignment, empty-state disappears and `<ResumenTable>` mounts.

5. **R5 — Dialog functional contract byte-identical**: Save / Load / Export / Import / Share flows behave exactly as Phase 8 shipped.
   - Current: `onExport` projects → `downloadBuildAsJson`; `onShare` projects → `encodeSharePayload` → URL-budget check → clipboard copy with JSON fallback; `onImportFile` parses → `diffRuleset` → `VersionMismatchDialog` OR `hydrateBuildDocument`; Save/Load mount `SaveSlotDialog` / `LoadSlotDialog`. `IncompleteBuildError` raises a toast via `shellCopyEs.persistence.incompleteBuild`.
   - Target: No change to any of the imported persistence helpers, no change to `SaveSlotDialog` / `LoadSlotDialog` bodies, no change to `VersionMismatchDialog` behavior, no change to `shellCopyEs.persistence.*` strings, no change to the `isProjectable` gating semantics. The five action buttons keep their exact `disabled`, `onClick`, `title`, and `variant` assignments.
   - Acceptance: `git diff apps/planner/src/features/persistence/**` returns empty for this phase's plan commits; `git diff apps/planner/src/features/summary/save-slot-dialog.tsx` returns empty; all existing Phase 8 / SHAR-* tests (`tests/phase-08/**` + any persistence tests under `tests/phase-12.*/` referencing persistence helpers) stay green without modification.

6. **R6 — Selector surface minimal**: `ResumenViewModel` + `resumen-selectors.ts` change surface is limited to what R1..R4 require.
   - Current: `useResumenViewModel` computes `identity`, `attributes[]`, `progression[]`, `skills[]`. `<ResumenTable>` consumes all four. `<ResumenBoard>` consumes only `identity.name`.
   - Target: `attributes[]` is either deleted from `ResumenViewModel` OR left in place as dead consumer-side (author's call during plan-phase). `identity`, `progression[]`, `skills[]` are unchanged in shape and computation. NO changes to `@rules-engine/*` helpers (`computeTotalBab`, `computeFortSave`, `computeRefSave`, `computeWillSave`). NO changes to compiled catalogs.
   - Acceptance: `git diff packages/rules-engine/` is empty for this phase's plan commits; `git diff apps/planner/src/data/compiled-*.ts` is empty; any `ResumenViewModel` shape change is scoped to `apps/planner/src/features/summary/resumen-selectors.ts` and its direct consumers.
