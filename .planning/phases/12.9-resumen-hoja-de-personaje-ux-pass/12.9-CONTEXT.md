<decisions>
## Implementation Decisions

### Layout (GA-1, GA-2)

- **D-01 Progresión full-width mechanism**: A BEM-style modifier class `.resumen-table__block--progresion` receives `grid-column: 1 / -1` to break the block out of the existing `.resumen-table` auto-fit grid. `.resumen-table` keeps its `grid-template-columns: repeat(auto-fit, minmax(18rem, 1fr))` rule intact — the modifier overrides column span on Progresión only. Minimal change; no CSS refactor; reuses the existing grid engine. Identity-header + Habilidades stay inside the auto-fit.

- **D-02 Habilidades full-width**: `.resumen-table__block--habilidades` also receives `grid-column: 1 / -1`. Reasoning: 39 skills × 4 cols still cramps an 18rem track; making both data-heavy blocks full-width simplifies the mental model (identity-header stacks on top, then two full-width tables). No CSS `columns: 2` multi-column trick — would regress accessibility (TH/TD row associations break across CSS column breaks) and falls outside the "no new layout primitives" constraint.

- **D-03 Identity-header replacement**: The existing `<dl>.resumen-table__identity` + `<table>.resumen-table__attrs` are BOTH deleted from `<ResumenTable>` and replaced by a single `<header>.resumen-table__identity-header` element rendering the compact line `{name} · {race}{subrace?} · {alignment} · {dataset}` (middot `·` U+00B7 separators, single `color: var(--color-gold-faded)` span wrapping the dataset piece so it picks up the same faded tone already used by `.resumen-board__dataset`). No new labels (no "Nombre:" prefix), no `<dl>` structure — the stepper summary bar already provides labeled identity; this is the ficha-header surface.

### Empty-state (GA-4)

- **D-04 Empty-state voice + shape**: Pre-race/alignment, `<ResumenBoard>` mounts a single `.resumen-board__empty-state.nwn-frame` with a bespoke heading + body (new `shellCopyEs.resumen.emptyState.heading` + `shellCopyEs.resumen.emptyState.body` keys). No CTA button — the stepper is always visible on the left, the user can navigate back unassisted. Copy voice is ficha/handoff-oriented (`"Todavía no hay ficha que mostrar"` / body routes user to complete raza + alineamiento en Construcción). No reuse of `stepper.emptySheetHeading` — different surface, different voice.

### Spacing (R3)

- **D-05 Inner-padding baseline adoption**: `.resumen-table__block` keeps its current outer padding but normalizes inner block-heading spacing by adopting the Dotes 8px baseline. Concretely: `.resumen-table__heading` gains `padding: 0 var(--space-xs, 0.5rem)` and `.resumen-board__actions` adjusts from `padding: 0.75rem 1rem` to `padding: var(--space-sm, 0.75rem) var(--space-xs, 0.5rem)` (matches `.feat-board__section-header` rhythm). Prefer existing tokens if they exist in `:root`; fall back to literal values that match the Dotes sweep. ZERO new tokens introduced; researcher to confirm exact existing token names before executor writes the CSS.

### Style Invariants (Phase 12.8 carry-over)

- **D-06 No new tokens / hex / font-weight**: No new CSS custom properties, no new hex colors, no new `font-weight: 700` declarations. Reuse `var(--color-gold)`, `var(--color-gold-faded)`, `var(--color-panel)`, `.nwn-frame`, existing `var(--space-*)` tokens. Style invariant inherited from Phase 12.8 (see `.planning/STATE.md` Decisions D-12.4-07/08/09/12.7-03 chain). Grep gate: `git diff apps/planner/src/styles/app.css` must produce zero new hex triplets + zero new `font-weight: 700` declarations.

### Persistence Contract (R5)

- **D-07 Dialog contract byte-identical**: `resumen-board.tsx` action handlers (`onExport` / `onShare` / `onImportFile`) + SaveSlot/LoadSlot/VersionMismatch dialog mounts stay byte-identical. Plan commits MUST leave `git diff apps/planner/src/features/persistence/**` + `git diff apps/planner/src/features/summary/save-slot-dialog.tsx` empty. Only `resumen-board.tsx` JSX around the action bar + empty-state conditional may change; handler bodies do not. The `isProjectable` predicate and its usage are preserved.

### Selector Surface (R6)

- **D-08 `resumen-selectors.ts` minimal delta**: `ResumenViewModel.attributes` is deleted because `<ResumenTable>` no longer consumes it (D-03 drops the attrs table). `attributes: AttributeRow[]` field + the `ATTRIBUTES_IN_ORDER.map(...)` block that builds it + the `applyRaceModifiers` / `computeModifier` helpers local to this selector are all removed if no other consumer exists (grep-verified). `identity`, `progression[]`, `skills[]` stay unchanged in shape and computation. ZERO changes to `@rules-engine/*` helpers (`computeTotalBab`, `computeFortSave`, `computeRefSave`, `computeWillSave`) — researcher/planner must confirm no callers outside this file reference `ResumenViewModel.attributes`.

### Testing (R1 / R2 / R4)

- **D-09 Test authoring convention**: New specs live under `tests/phase-12.9/` using Vitest RTL. Follow Phase 12.8-03 D-13 convention: `createElement(Component, props)` NOT JSX in `.spec.tsx` files (Vitest default esbuild does not auto-inject React runtime) + explicit `afterEach(cleanup)`. File split: one spec per requirement — `resumen-progresion-full-width.spec.tsx` (R1), `resumen-identity-dedup.spec.tsx` (R2), `resumen-empty-state.spec.tsx` (R4). R3 (spacing) is covered by a grep-based assertion inside the empty-state spec file OR a dedicated mini-suite — planner's call. R5 + R6 do not need new specs; the existing Phase 8 + Phase 12.x suites act as regression guards, kept green by the zero-diff invariant.

- **D-10 jsdom-only coverage**: No Playwright required for this phase. The 8-col column-fit check (R1) is handled at jsdom layout measurement via `scrollWidth <= clientWidth` assertion. Live MCP Chrome UAT against the locked fixture (Elfo + Neutral puro + Guerrero L1..L20 @ localhost:5173) is the final verification gate, documented in `12.9-VERIFICATION.md` at phase close.
