# Quick Task 260419-68b: Fix Atributos Board Misalignment

## Problem

In-browser UAT (localhost:5173, route `/construccion/atributos`) surfaced three UI defects on `AttributesBoard`:

1. **Column misalignment (primary).** `.attributes-editor__row` uses `display: flex` with `.attributes-editor__label { flex: 1 }` and per-cell `min-width: 72px`. Base cell `[-] <value> [+]` actually renders ~140px (two 44px-min buttons + value), overflowing the 72px header column. Racial/final cells get pushed rightward by variable amounts per row (CON `+2` pushes further than FUE `0`; INT FINAL clips behind right-hand `DetailPanel`). Column headers (`Base/Racial/Final`) line up with themselves but not with data rows.
2. **`-` button glyph near-invisible.** `.nwn-button--secondary` uses `color: var(--color-text-muted)` at 14px — dash barely readable against dark panel background on screenshot.
3. **Old-style numerals on attribute values.** `--font-display` defaults to oldstyle-nums, so `10` renders as `io`, `0` as `o`. Legible once identified but user-hostile for a numeric editor.

## Goal

Attributes editor layout aligns across all six rows (FUE/DES/CON/INT/SAB/CAR) with header cells; all data fully visible inside left panel; +/- buttons show clear glyphs; numeric values read as lining digits (`0`, `10`, not `o`, `io`).

## Scope

`apps/planner/src/styles/app.css` — CSS-only patch. No JSX changes. No test changes (pure visual).

## Tasks

### Task 1 — Grid-based row alignment

**Files:** `apps/planner/src/styles/app.css` (lines ~108-122 — `.attributes-editor__row`, `.attributes-editor__column-headers`, `.attributes-editor__cell*`, `.attributes-editor__label`, `.attributes-editor__mod`).

**Action:**
- Replace `.attributes-editor__row { display: flex; ... }` with `display: grid` + `grid-template-columns: minmax(0, 1fr) 140px 64px 56px 48px`.
- Apply same `grid-template-columns` to `.attributes-editor__column-headers`.
- Remove `flex: 1` from `.attributes-editor__label` and `.attributes-editor__column-headers-label` (grid handles sizing).
- Remove `min-width: 72px` from `.attributes-editor__cell` (grid cell already constrains).
- Keep `.attributes-editor__cell--base` as inline-flex internally for the button row; grid cell contains it.
- Add `justify-self: center` / `text-align: center` on racial/final cells, `justify-self: end` / `text-align: right` on mod cell, to keep alignment after grid takeover.

**Verify:**
- Screenshot at viewport 1280×900: FUE/DES/CON/INT/SAB/CAR rows align column-for-column with Base/Racial/Final header.
- CON row `+2` does not shift FINAL column rightward.
- INT row FINAL value fully visible inside left panel (no clip behind right `DetailPanel`).

**Done when:** All six rows share identical x-positions for Base/Racial/Final/Mod columns.

### Task 2 — Button glyph visibility

**Files:** `apps/planner/src/styles/app.css`.

**Action:**
- Scoped override inside `.attributes-editor__cell--base .nwn-button` (not global `.nwn-button--secondary`, to avoid regressing other secondary buttons): bump `color: var(--color-text)`, `font-weight: 700`, `font-size: 18px`, and `min-height: 36px` with `padding: var(--space-xs) var(--space-sm)` so +/- buttons stay compact + readable.

**Verify:**
- `-` and `+` glyphs render at equal weight + contrast to attribute value.
- Disabled state still dims via existing `.nwn-button:disabled { opacity: 0.45 }`.

**Done when:** Both button glyphs legible in screenshot; disabled rows (e.g. FUE at `-` when base=8=minimum) visibly dimmer but glyph still visible.

### Task 3 — Lining numerals on attribute values

**Files:** `apps/planner/src/styles/app.css`.

**Action:**
- Add `font-variant-numeric: lining-nums tabular-nums;` to `.attributes-editor__value`, `.attributes-editor__cell--racial`, `.attributes-editor__cell--final`, `.attributes-editor__mod`, and the header `.attributes-editor__header` span (so `Puntos restantes: 30` reads correctly too).
- Tabular-nums locks digit advance so base/racial/final columns don't jitter when values change `9 → 10`.

**Verify:**
- CON racial shows `+2` not `+two-like-glyph`; FUE final shows `10` not `io` after a `+` click.
- `Puntos restantes: 30` reads as lining `30`, not `3o`.

**Done when:** All numeric rendering uses lining digits.

- Other routes / components using `.nwn-button--secondary` — scoped override prevents blast-radius.
