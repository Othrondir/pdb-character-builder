# Objetivo

Reducir la deuda visible de la suite Vitest eliminando `skipped`/`todo` que no deben aparecer en la ejecución normal:

- Hacer configurables por entorno las rutas locales del extractor NWN/Puerta.
- Evitar que las suites de integración `phase-05.1` cuenten como `skipped` cuando los datos propietarios no están presentes.
- Reescribir o retirar tests obsoletos marcados con `describe.skip`.
- Convertir el `it.todo` de presupuesto racial en una expectativa explícita del estado fuente actual.

# Validación

- `vitest` focalizado sobre los specs tocados.
- `tsc -p tsconfig.base.json --noEmit`.
- Suite completa si el tiempo lo permite.

# Resultado

- Las rutas del extractor aceptan variables de entorno y defaults por plataforma:
  - Linux: `~/.local/share/Neverwinter Nights/nwsync`
  - Linux Steam: `~/.local/share/Steam/steamapps/common/Neverwinter Nights`
  - Windows: se preservan los defaults previos.
- Las integraciones `phase-05.1` ya no se registran como `skipped`; en este entorno se ejecutan con datos reales.
- El `describe.skip` de progresión se convirtió en contrato activo del título estático `Progresión 1-20`.
- El `it.todo` de presupuesto racial se convirtió en una aserción explícita de uniformidad, coherente con los datos fuente actuales.
- El `it.skip` defensivo de clases base ahora falla en rojo si falta una clase del catálogo.
