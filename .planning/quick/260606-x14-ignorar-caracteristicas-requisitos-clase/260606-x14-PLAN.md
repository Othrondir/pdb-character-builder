# Quick Plan: Caracteristicas no bloquean clases

## Objetivo

Permitir seleccionar clases aunque el personaje no tenga una caracteristica minima aparente. Las clases deben bloquearse solo por requisitos reales de clase como alineamiento, deidad, dotes, habilidades, BAB, nivel de clase o reglas de multiclase; las caracteristicas no son requisitos de seleccion.

## Pasos

1. Cambiar `evaluateClassEntry` para ignorar `minimumAbilityScores`.
2. Quitar la INT minima del overlay runtime de Mago.
3. Actualizar tests de requisitos de clase con Mago, Paladin, Druida, Hechicero y Brujo con caracteristica baja seleccionables cuando cumplen el resto de requisitos.
4. Ejecutar Vitest enfocado y typecheck.
