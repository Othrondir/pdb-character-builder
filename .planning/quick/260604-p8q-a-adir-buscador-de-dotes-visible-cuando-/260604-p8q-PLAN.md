# Quick Task 260604-p8q: Añadir buscador de dotes y hacer explícito Quitar

Date: 2026-06-04
Status: planned

## Goal

When the feat picker is active, show a visible search box that only filters by
name from 3 characters onward, ignoring case and accents, keeping the full list
for shorter queries, and exposing a clear action. Also make deselection more
discoverable by surfacing an explicit `Quitar` action on already-selected rows
without removing the existing click-to-deselect behavior or the folded feat
families.

## Tasks

1. **Wire the visible feat search UI in the picker**
   - Files: `apps/planner/src/features/feats/feat-sheet.tsx`, `apps/planner/src/lib/copy/es.ts`, optionally `apps/planner/src/styles/app.css` only if the existing `.feat-board__search*` rules need a minimal adjustment.
   - Add a search input inside the Dotes picker surface (not hidden in the detail panel), plus a visible clear control.
   - Keep the current full list when the normalized query has fewer than 3 characters; start filtering only at 3+.
   - Normalize both query and feat labels case-insensitively and accent-insensitively.
   - Preserve the current section split (clase / racial / general, available / unavailable) and do not break folded families while filtering.

2. **Make selected rows advertise deselection with an explicit `Quitar` action**
   - Files: `apps/planner/src/features/feats/feat-sheet.tsx`, `apps/planner/src/lib/copy/es.ts`, `apps/planner/src/styles/app.css`.
   - For rows already chosen at the active level, surface a clear `Quitar` affordance directly in the row/family row while keeping the existing row-click deselect behavior intact.
   - Apply the same discoverability to folded families when the selected state lives inside the family entry, without unfolding the family model or changing how the stored feat id works.
   - Keep this work scoped around the existing local edits in `selectors.ts` and `tests/phase-12.4/feat-selectability-states.spec.tsx`; do not revert or re-scope unrelated local changes.

3. **Lock the behavior with focused feat-picker tests**
   - Files: `tests/phase-12.4/feat-family-expander.spec.tsx`, `tests/phase-12.4/feat-selectability-states.spec.tsx`, and/or one new focused feat-picker spec if that is cleaner than overloading the existing files.
   - Cover: visible search box, 0-2 chars = unfiltered list, 3+ chars = filtered list, accent/case-insensitive matching, clear action restoring the full list, explicit `Quitar` affordance on selected simple rows, and explicit `Quitar` affordance on selected family rows.
   - Verify with focused suites plus typecheck, e.g. `corepack pnpm exec vitest run tests/phase-12.4/feat-selectability-states.spec.tsx tests/phase-12.4/feat-family-expander.spec.tsx` and `corepack pnpm exec tsc -p tsconfig.base.json --noEmit`.
