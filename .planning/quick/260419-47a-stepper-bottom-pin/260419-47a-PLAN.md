# Quick Task 260419-47a — Pin Stepper Bottom

## Problem

Aside `.creation-stepper` at `apps/planner/src/styles/app.css:985` uses:

```css
.creation-stepper { display: flex; flex-direction: column; height: 100%; overflow-y: auto; }
.creation-stepper__bottom { margin-top: auto; ... }
```

When the progression rail + expanded level sub-steps push content past the viewport height, the outer `overflow-y: auto` lets content scroll — but `.creation-stepper__bottom` is a flex child living inside the scroll container, so `Resumen` and `Utilidades` buttons get scrolled out of view below the fold. User UAT 2026-04-19 surfaced: *"el botón de utilidades se sale del div"*.

## Goal

`Resumen` + `Utilidades` buttons stay pinned at the bottom of the aside regardless of progression content length. Progression content (level rail + expanded sub-steps) scrolls internally without pushing the bottom buttons out of view.

## Scope

CSS-only change to `apps/planner/src/styles/app.css`. No JSX changes. No test changes.

## Tasks

### Task 1 — Grid layout stepper with pinned bottom

**Files:** `apps/planner/src/styles/app.css` — `.creation-stepper` block at lines 985+, `.creation-stepper__progression` block at lines 1007-1008.

**Action:**
- Change `.creation-stepper` from `display: flex; flex-direction: column; overflow-y: auto` to `display: grid; grid-template-rows: auto auto 1fr auto; height: 100%; overflow: hidden`.
- Grid row assignment (natural order of JSX children in `creation-stepper.tsx`): row 1 = `.creation-stepper__title-bar`, row 2 = `.creation-stepper__origin`, row 3 = `.creation-stepper__progression` (takes remaining `1fr`), row 4 = `.creation-stepper__bottom` (pinned last auto row).
- Add `min-height: 0` + `overflow-y: auto` to `.creation-stepper__progression` so its content scrolls internally without blowing out the grid cell.
- Keep `margin-top: auto` removed from `.creation-stepper__bottom` (no longer needed under grid — position inherent to row 4).

**Verify:**
- At viewport 1280×900 with L1..L16 rail expanded + sub-steps visible, `Resumen` and `Utilidades` buttons remain visible at the bottom of the aside.
- Progression content above can scroll independently.
- Scrollbar lives on `.creation-stepper__progression`, not on the outer stepper frame.

**Done when:** Both bottom buttons visible in screenshot after reload; internal progression scroll works.
